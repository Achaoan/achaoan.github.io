动态代理的实现两种方式和区别

本文最后更新于：12 分钟前

一、JDK 动态代理

基于 Java 反射机制，运行时动态生成实现接口的代理类。

核心类/接口

  • java.lang.reflect.Proxy：创建代理对象
  • java.lang.reflect.InvocationHandler：调用处理器接口

必要条件

目标类必须实现至少一个接口。

实现示例

原理

代理类继承 Proxy 类，实现目标类的所有接口。调用代理方法时，会触发 InvocationHandler.invoke()，通过反射调用目标方法。

二、CGLIB 动态代理

基于 字节码技术（ASM），运行时动态生成目标类的子类。

核心类

  • net.sf.cglib.proxy.Enhancer：创建代理对象
  • net.sf.cglib.proxy.MethodInterceptor：方法拦截器

必要条件

目标类不能是 final 类，方法不能是 final。

实现示例

原理

CGLIB 使用 ASM 字节码框架生成目标类的子类，重写其所有非 final 方法。调用代理方法时触发 MethodInterceptor.intercept()，通过 MethodProxy.invokeSuper() 调用目标类的原始方法。

三、核心区别对比

对比项 JDK 动态代理 CGLIB 动态代理
对接口要求 必须实现接口 不要求
底层技术 反射机制 ASM 字节码操作
创建代理速度 快 较慢
运行时执行速度 较慢（反射调用） 更快（直接调用）
代理限制 只代理接口方法 不能代理 final 类/方法
依赖 JDK 内置，零依赖 需引入 CGLIB（spring-core 已内置）

四、Spring AOP 中的选择策略

五、如何选择？

场景 推荐
目标类实现了接口，对性能要求高 JDK 动态代理
目标类没有实现接口 CGLIB
追求运行时性能 CGLIB
追求创建代理速度 JDK 动态代理
希望零依赖 JDK 动态代理

一句话：JDK 代理面向接口，CGLIB 代理面向类；JDK 创建快、执行慢，CGLIB 创建慢、执行快。
