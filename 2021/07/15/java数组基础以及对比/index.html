数组基础

数组是一种线性数据结构，用连续的内存空间存储相同类型的数据。Java 中数组是对象，直接继承自 Object。

特点：

  • 长度固定，一旦创建不可变
  • 内存连续，支持随机访问 O(1)
  • 插入和删除需要移动元素 O(n)

3. HashMap 源码分析

底层结构（JDK 1.8+）： 数组 + 链表 + 红黑树

特性 值
初始容量 16（懒加载）
负载因子 0.75
扩容倍数 2 倍
树化阈值 链表长度 ≥ 8 且数组长度 ≥ 64
反树化阈值 红黑树节点 ≤ 6

HashMap 的 put 流程：

  1. 数组为空 → resize() 初始化
  2. 计算 hash → (h = key.hashCode()) ^ (h >>> 16) 扰动函数
  3. 计算下标 → (n - 1) & hash
  4. 若下标无元素 → 直接插入
  5. 若有元素：
    • key 相同 → 覆盖 value
    • 红黑树 → 按红黑树规则插入
    • 链表 → 尾插法插入；长度 ≥ 8 且数组 ≥ 64 → 树化
  6. 元素数量 > threshold → resize() 扩容

为什么容量必须是 2 的幂次方？

  • (n-1) & hash 等价于取模，位运算更快
  • 扩容时通过 hash & oldCap == 0 判断元素位置

JDK 1.7 vs 1.8 区别：

  • 1.7：数组 + 链表，头插法，多线程扩容可能死循环
  • 1.8：数组 + 链表 + 红黑树，尾插法，解决死循环

面试题精选

1. 为什么用 HashMap？

HashMap 实现了 O(1) 的插入、删除、查找的高性能 KV 存储。相比 ArrayList（查找快但增删慢）和 LinkedList（增删快但查找慢），HashMap 综合性能最均衡。

2. HashMap 是不是线程安全的？为什么？

不是。多线程操作时，扩容可能造成循环链表（JDK 1.7），数据覆盖等问题。需改用 ConcurrentHashMap 或 Collections.synchronizedMap()。

3. 为什么负载因子是 0.75？

空间与查询效率的平衡。基于泊松分布（Poisson Distribution），在合理哈希函数下冲突概率最低。

4. 为什么链表转红黑树的阈值是 8？

根据泊松分布，链表长度达到 8 的概率仅为 0.00000006（亿分之六），红黑树作为保险机制防止极端情况性能崩塌。

5. HashSet 与 HashMap 的关系？

HashSet 底层基于 HashMap 实现，元素作为 HashMap 的 key，value 统一为 PRESENT 占位对象。
