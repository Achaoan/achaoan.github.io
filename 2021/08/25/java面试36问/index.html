1. 谈谈你对 Java 平台的理解？”Java 是解释执行”，这句话正确吗？

Java 是一种面向对象的编程语言，具备跨平台能力（Write Once, Run Anywhere）。Java 源码首先被编译为字节码（bytecode），然后在 JVM 上解释执行或 JIT 编译执行。

“Java 是解释执行”不完全正确：

  • Java 源码经过 javac 编译成字节码（.class 文件）
  • JVM 启动时，热点代码通过 JIT（Just-In-Time）编译器 编译为本地机器码直接执行
  • 非热点代码通过解释器逐行解释执行
  • 所以 Java 是 编译 + 解释 混合执行模式

2. Exception 和 Error 的区别？

  • Exception：程序可处理的异常，如 IOException、SQLException
  • Error：程序无法处理的严重问题，如 OutOfMemoryError、StackOverflowError
  • Exception 分为 Checked Exception（编译时异常，必须处理）和 Unchecked Exception（运行时异常，可不用处理）

3. final、finally、finalize 的区别？

关键字 用途
final 修饰类（不可继承）、方法（不可重写）、变量（不可变）
finally try-catch 中保证一定执行的代码块（释放资源）
finalize Object 的方法，GC 回收前调用（不推荐使用，Java 9 已弃用）

4. 强引用、软引用、弱引用、虚引用的区别？

类型 回收时机 使用场景
强引用 OOM 也不回收 普通 new 对象
软引用 内存不足时回收 内存敏感缓存
弱引用 下次 GC 即回收 WeakHashMap、ThreadLocal
虚引用 随时可能回收 跟踪对象回收

5. String、StringBuilder、StringBuffer 的区别？

类 可变性 线程安全 性能
String 不可变 安全 拼接时产生新对象
StringBuilder 可变 不安全 最快
StringBuffer 可变 安全（synchronized） 较慢

使用原则： 少量字符串拼接用 +；单线程循环拼接用 StringBuilder；多线程用 StringBuffer。

6. 重载（Overload）和重写（Override）的区别？

维度 重载 重写
位置 同类中 子类中
参数 必须不同 必须相同
返回类型 可不同 必须相同
访问权限 不限 不能更严格
绑定时机 编译期 运行期

7. 接口和抽象类的区别？

  • 抽象类：is-a 关系，代码复用，单继承，有构造方法，可有任意成员变量
  • 接口：like-a 关系，行为契约，多实现，无构造方法，变量默认 public static final

8. == 和 equals 的区别？

  • ==：基本类型比较值，引用类型比较内存地址
  • equals()：Object 中默认同 ==，String、Integer 等类已重写为比较内容

9. hashCode() 和 equals() 的约定？

  • 两个对象 equals 相等 → hashCode 必须相等
  • 两个对象 hashCode 相等 → equals 不一定相等（哈希冲突）
  • 重写 equals 必须重写 hashCode，否则 HashMap 等集合会出问题

10. ArrayList 和 LinkedList 的区别？

ArrayList 基于数组（连续内存，查询快 O(1)），LinkedList 基于双向链表（增删快 O(1)）。99% 场景用 ArrayList。

11. HashMap 的实现原理？

JDK 1.8：数组 + 链表 + 红黑树。通过 (n-1) & hash 计算下标。当链表长度 ≥ 8 且数组 ≥ 64 时树化。初始容量 16，负载因子 0.75，扩容 2 倍。

12. ConcurrentHashMap 如何保证线程安全？

JDK 1.8：CAS + synchronized，锁粒度细化到单个桶，性能大幅提升。

13. 什么是线程安全？

多个线程同时访问共享资源时，无论执行顺序如何，结果都正确。可通过 synchronized、Lock、原子类、volatile、ThreadLocal 等实现。

14. synchronized 的实现原理？

基于 JVM 的内置锁（Monitor），JDK 1.6 后引入了偏向锁 → 轻量级锁 → 重量级锁的升级过程，减少了锁竞争的开销。

15. volatile 关键字的作用？

保证可见性（修改立即写主存）和禁止指令重排序（内存屏障），但不保证原子性。

16. ThreadLocal 的原理和内存泄漏？

每个线程维护一个 ThreadLocalMap，key 是 ThreadLocal 弱引用。内存泄漏原因：key 被回收后 value 还存在。解决方案：用完调用 remove()。

17. 线程池的核心参数有哪些？

corePoolSize、maximumPoolSize、keepAliveTime、unit、workQueue、threadFactory、handler。

18. 线程池的拒绝策略有哪些？

AbortPolicy（抛异常）、CallerRunsPolicy（调用者执行）、DiscardPolicy（丢弃）、DiscardOldestPolicy（丢弃最老任务）。

19. JVM 内存区域有哪些？

堆、方法区（元空间）、虚拟机栈、本地方法栈、程序计数器。其中堆和方法区是线程共享，其他是线程私有。

20. 垃圾回收算法有哪些？

标记-清除、复制算法、标记-整理、分代收集。新生代用复制算法，老年代用标记-整理/清除。

21. CMS 和 G1 的区别？

CMS 基于标记-清除，有内存碎片；G1 分区管理，可预测停顿时间，推荐大内存场景（>4G）。

22. 什么是类加载的双亲委派模型？

一个类加载器收到加载请求后，先委派给父加载器，父加载器无法加载时再自己加载。保证核心类库不会被替换。

23. 什么是代理模式？JDK 动态代理和 CGLIB 的区别？

代理模式提供了对目标对象的间接访问。JDK 动态代理基于接口（反射），CGLIB 基于继承（字节码）。参见《动态代理的实现两种方式和区别》。

24. 什么是反射？有什么应用场景？

反射是运行时动态获取类的信息并操作的能力。应用：Spring IoC、动态代理、ORM 框架、注解解析。

25. 什么是序列化？什么时候需要？

将 Java 对象转为字节流便于存储或传输。需要时：RPC 通信、Redis 存储对象、文件持久化。实现 Serializable 接口。

26. try-catch-finally 中，finally 一定会执行吗？

不一定：System.exit() 时、JVM 崩溃时、无限循环时不会执行。

27. 常见的运行时异常有哪些？

NullPointerException、ArrayIndexOutOfBoundsException、ClassCastException、IllegalArgumentException、ArithmeticException。

28. BIO、NIO、AIO 的区别？

  • BIO：阻塞 IO，一个连接一个线程
  • NIO：非阻塞 IO，多路复用，一个线程管理多个连接
  • AIO：异步 IO，回调机制

29. 什么是泛型？有什么作用？

泛型提供了编译时类型安全检测机制。作用：类型安全、消除强制类型转换、代码复用。

30. 什么是注解？

注解是代码中的元数据，可以被编译时或运行时解析。如 @Override、@Transactional。

31. 什么是 Lambda 表达式？

Java 8 引入的函数式编程特性，简化匿名内部类的写法。(参数) -> { 方法体 }。

32. Stream API 了解吗？

Java 8 引入的流式处理 API，支持链式操作：filter、map、reduce、collect、sorted 等。

33. Optional 是什么？

Java 8 引入的容器类，用于解决 NullPointerException 问题。常用方法：ofNullable、orElse、ifPresent。

34. 什么是 SPI 机制？

SPI（Service Provider Interface）是 JDK 内置的服务发现机制，允许第三方提供接口的实现。应用：JDBC 驱动加载。

35. 什么是零拷贝？

减少或避免数据在内核空间和用户空间之间的拷贝。如 Kafka、Netty 利用 FileChannel.transferTo() 实现零拷贝。

36. 你了解哪些 Java 设计模式？

单例（Double Check Lock）、工厂（BeanFactory）、代理（AOP）、观察者（事件监听）、模板（JdbcTemplate）、策略（Comparator）、责任链（Filter）。
