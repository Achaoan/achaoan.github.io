Spring基础及相关问题

本文最后更新于：3 小时前

一、IoC（控制反转）

概念

IoC（Inversion of Control）控制反转，对象的创建权从业务代码转移到 Spring 容器。容器通过反射机制根据配置或注解动态创建对象。

DI（依赖注入）

DI 是 IoC 的具体实现，有三种方式：

  1. 构造器注入：通过构造方法注入依赖
  2. Setter 注入：通过 setter 方法注入
  3. 注解注入：通过 @Autowired、@Resource 等注解注入

底层原理

工厂模式 + 反射：

Spring 容器

  • BeanFactory：最底层容器，延迟实例化
  • ApplicationContext：高级容器，扩展了事件、国际化等功能，默认启动时实例化所有单例 Bean

二、AOP（面向切面编程）

概念

AOP 将日志、事务、权限等横切逻辑从业务代码中抽离为”切面”，实现关注点分离。

核心术语

术语 说明
JoinPoint 连接点，程序执行的某个特定位置（如方法调用）
Pointcut 切点，匹配连接点的表达式
Advice 增强，在切点执行的逻辑（Before、After、Around 等）
Aspect 切面 = Pointcut + Advice
Weaving 织入，将切面应用到目标对象的过程

通知类型

  • @Before：方法执行前
  • @After：方法执行后（无论是否异常）
  • @AfterReturning：方法正常返回后
  • @AfterThrowing：方法抛出异常后
  • @Around：环绕通知，最强大

底层原理

AOP 底层依赖动态代理：

  • JDK 动态代理：基于接口，目标类必须实现接口
  • CGLIB 动态代理：基于继承，目标类不能是 final

三、静态代理和动态代理

静态代理

手动编写代理类，每个目标类对应一个代理类，编译期已确定。

动态代理

在程序运行时，运用反射机制动态创建对象实例。

JDK 动态代理必须具备的条件：

  1. 目标接口
  2. 目标类
  3. InvocationHandler（拦截器）
  4. 代理类

Spring 的选择规则：

  • 对于有接口的类 → 使用 JDK Proxy
  • 对于没有接口的类 → 使用 CGLIB
  • 虽然都可以用 CGLIB，但 CGLIB 复杂、效率低，且不能代理 final 修饰的类或方法

四、反射（Reflection）

在运行过程中，可以对任意类加载该对象的所有属性和方法，并调用它们。这种动态获取类的信息和动态调用类的方法的机制就是反射机制。

核心 API

使用场景

  • Spring IoC 容器创建 Bean
  • JDK 动态代理调用目标方法
  • 注解解析（如 @Transactional）
  • ORM 框架结果映射
  • JSON 序列化/反序列化

五、@Autowired 和 @Resource 区别

注解 装配方式 来源 匹配规则
@Autowired 按类型装配 Spring byType，配合 @Qualifier 可按名称
@Resource 按名称装配 JDK（JSR-250） byName，可通过 name 属性指定

使用建议：

  • @Autowired 需要 required 属性控制是否必须
  • @Resource 默认按名称，适合明确 bean name 的场景

六、自动装配

手动装配

以 value 或 ref 的方式明确指定属性值。

自动装配

根据指定的装配规则，Spring 自动将匹配的属性值注入 bean 中（仅适用于注入对象）。

装配模式：

  1. 根据类型自动装配：将类型匹配的 bean 注入到另一个 bean 中。如果 IOC 容器中有多个同类型的 bean，Spring 将无法注入，抛出异常。
  2. 根据名称自动装配：必须将目标 bean 的名称和属性名设置完全相同。
  3. 通过构造器自动装配：当 bean 中存在多个构造器时使用，不建议。

七、Bean 生命周期

  1. 实例化（反射创建对象）
  2. 属性赋值（依赖注入）
  3. 处理 Aware 接口（BeanNameAware、ApplicationContextAware 等）
  4. BeanPostProcessor#postProcessBeforeInitialization
  5. InitializingBean#afterPropertiesSet / @PostConstruct / init-method
  6. BeanPostProcessor#postProcessAfterInitialization
  7. Bean 使用中
  8. DisposableBean#destroy / @PreDestroy / destroy-method

八、循环依赖（三级缓存）

Spring 通过 三级缓存 解决 Setter 注入的循环依赖：

  • 一级缓存 singletonObjects：完整 Bean
  • 二级缓存 earlySingletonObjects：半成品，提前暴露
  • 三级缓存 singletonFactories：ObjectFactory，延迟生成代理

构造器注入的循环依赖无法解决。

九、Spring 中用到的设计模式

设计模式 应用场景
工厂模式 BeanFactory
单例模式 Bean 默认作用域
代理模式 AOP、声明式事务
模板方法 JdbcTemplate、RestTemplate
观察者模式 事件驱动（ApplicationEvent）
适配器模式 MVC HandlerAdapter

  6. Spring 事务传播行为有哪些？

    • 异常被 catch 吞掉

    • propagation 设置错误
  8. Spring Bean 的作用域
    • singleton、prototype、request、session、application
