Elasticsearch基础及相关问题

本文最后更新于：1 小时前

一、Elasticsearch 简介

Elasticsearch（ES）是一个基于 Lucene 的分布式 RESTful 实时搜索和分析引擎。每个字段都被索引并可被搜索，可扩展至上百台服务器，处理 PB 级数据。具备近实时（NRT）特性：数据写入后约 1 秒即可检索。

核心概念（与关系型数据库对比）

Elasticsearch 关系型数据库
索引（Index） 数据库
文档（Document） 行
字段（Field） 列
映射（Mapping） 表结构
分片（Shard） 分区

倒排索引（Inverted Index）

ES 的核心数据结构，将”关键词 → 文档 ID”建立映射，实现 O(1) 时间复杂度检索。底层基于 FST（Finite State Transducer） 实现，空间占用小、查询速度快。

text 与 keyword 类型区别

  • text：会分词，用于全文搜索，不用于排序/聚合
  • keyword：不分词，用于精确搜索、过滤、排序、聚合

二、工作原理

写入流程

  1. 客户端请求 → 协调节点（Coordinating Node）
  2. 协调节点通过路由算法计算目标分片：shard = hash(document_id) % primary_shards
  3. 请求转发到 Primary Shard
  4. Primary Shard 写入 Memory Buffer，同时写入 Translog
  5. 默认每秒 refresh：Memory Buffer → Filesystem Cache（此时可被搜索）
  6. 默认每 30 分钟或 Translog 超 512MB 触发 flush：数据持久化到磁盘

搜索流程（Query Then Fetch）

Query 阶段：

  • 协调节点将请求广播到所有分片（主分片或副本）
  • 每个分片本地搜索并构建优先队列（from + size）
  • 返回文档 ID 和排序值给协调节点

Fetch 阶段：

  • 协调节点合并排序，确定最终结果
  • 根据 doc ID 拉取完整文档数据
  • 返回结果给客户端

更新与删除

  • 文档不可变，删除是在 .del 文件中标记为 deleted
  • 更新 = 标记旧文档为 deleted + 创建新文档
  • 段合并时物理删除标记的文档

三、常见面试题

1. ES 如何保证读写一致性？

  • 乐观并发控制：使用 _version 版本号
  • 写一致性：支持 one / quorum（默认）/ all
  • 读一致性：可设置 _preference=primary 查询主分片

2. ES 集群如何调优？

索引层面：

  • 批量写入（5-15MB/批）
  • 写入前关闭副本（number_of_replicas: 0）
  • 调整 refresh_interval 到 30s
  • 使用 SSD、冷热分离

查询层面：

  • 避免深度分页（使用 search_after 或 scroll）
  • 多用 filter 上下文（可缓存）
  • 合理设置分片大小（20-40GB）

3. 什么是深度分页？如何解决？

深度分页会导致性能问题，ES 默认不支持超过 10000 条分页。解决方案：

  • scroll 滚动搜索：基于快照，适合导出数据
  • search_after：基于游标，适合实时分页

4. DocValues 的作用？

在构建倒排索引的同时构建正排索引（docId → 字段值），用于排序、聚合等操作。存储在磁盘（堆外内存），避免 OOM。

5. query 和 filter 的区别？

  • query：计算相关性评分（_score）
  • filter：仅判断是否匹配，不评分、结果可缓存（性能更好）

6. Master 选举与脑裂预防

  • 由 ZenDiscovery 模块负责
  • 候选节点根据 nodeId 字典排序
  • 得票数超过 (n/2)+1 且自己选自己即成为 Master
  • 脑裂预防：设置 discovery.zen.minimum_master_nodes = (N/2)+1
