JVM基础及回收算法

本文最后更新于：31 分钟前

一、JVM 内存区域

线程私有区域

  • 程序计数器：当前线程所执行的字节码行号指示器，记录下一条要执行的指令地址
  • 虚拟机栈：每个方法执行时创建栈帧，存储局部变量表、操作数栈、动态链接、方法出口
  • 本地方法栈：为虚拟机使用到的 Native 方法服务

线程共享区域

  • 堆（Heap）：存放对象实例，GC 管理的主要区域，可细分为新生代（Eden/S0/S1）+ 老年代
  • 方法区：存储类信息、常量、静态变量。JDK8 后改为元空间（Metaspace），使用本地内存

JDK8 永久代 → 元空间的变化

元空间使用本地内存，不再受 JVM 堆大小限制，避免 OOM。

二、对象存活判断

可达性分析（Java 使用）

从 GC Roots 开始向下搜索，不可达的对象判定为可回收。

可作为 GC Roots 的对象：

  • 虚拟机栈中引用的对象
  • 方法区静态属性引用的对象
  • 方法区常量池引用的对象
  • 本地方法栈 JNI 引用的对象

四种引用类型

类型 回收时机 用途
强引用 永不回收，OOM 也不回收 普通 new 对象
软引用 内存不足时回收 缓存（内存敏感场景）
弱引用 下次 GC 即回收 WeakHashMap、ThreadLocal
虚引用 任何时候都可能被回收 跟踪对象回收状态

三、垃圾回收算法

算法 原理 优点 缺点 适用场景
标记-清除 标记存活对象 → 清除未标记对象 实现简单 效率低、内存碎片 老年代（配合 CMS）
复制算法 将内存分为两块，存活对象复制到另一块 无碎片、高效 内存利用率低 新生代
标记-整理 标记存活对象 → 向一端移动 → 清理边界外 无碎片 移动对象有开销 老年代
分代收集 新生代用复制，老年代用标记-整理/清除 综合最优 实现复杂 主流 JVM 策略

四、触发机制

Minor GC（新生代 GC）

  • 触发条件：Eden 区满
  • 特点：非常频繁，回收速度快

Full GC（老年代 GC）

  • 触发条件：
    1. 老年代空间不足
    2. 元空间空间不足
    3. 调用 System.gc()
    4. Minor GC 后晋升对象大小 > 老年代可用空间
  • 特点：速度慢（是 Minor GC 的 10 倍以上），STW 时间长

对象晋升老年代的条件

  1. 年龄阈值：默认 15 岁（-XX:MaxTenuringThreshold）
  2. 动态年龄判定：Survivor 中相同年龄对象大小总和 > Survivor 空间的一半
  3. 大对象直接进入：通过 -XX:PretenureSizeThreshold 设置阈值

五、垃圾收集器

收集器 作用位置 算法 特点
Serial 新生代 复制 单线程，简单高效
ParNew 新生代 复制 多线程，配合 CMS
Parallel Scavenge 新生代 复制 吞吐量优先
CMS 老年代 标记-清除 低停顿
G1 全堆 标记-整理+复制 可预测停顿，大内存推荐
ZGC 全堆 并发 毫秒级停顿

CMS 流程

  1. 初始标记（STW）：标记 GC Roots 直接关联对象
  2. 并发标记：与用户线程并发，遍历对象图
  3. 重新标记（STW）：修正变动的对象
  4. 并发清除：与用户线程并发清除垃圾

G1 特点

  • 将堆划分为多个 Region（1-32MB），逻辑上分代
  • 维护优先列表，优先回收价值最大的 Region
  • 可设置期望 STW 时间（-XX:MaxGCPauseMillis）
  • 整体基于标记-整理，局部基于复制，无碎片

六、高频面试题

1. Minor GC 和 Full GC 区别？

Minor GC 在新生代频繁执行，速度快；Full GC 在老年代执行，速度慢，STW 时间长。

2. 新生代为什么用复制算法？

对象存活率低（约 98% 朝生夕死），复制成本低，不会产生内存碎片。

3. 什么是 Stop-The-World？

GC 时暂停所有应用线程，保证一致性快照。CMS 和 G1 通过并发设计尽量缩短 STW 时间。

4. 类加载过程？

加载 → 验证 → 准备 → 解析 → 初始化

5. 双亲委派模型？

先委派父加载器，父无法加载才自加载，保证核心类安全（不会出现多个版本的 java.lang.Object）。
