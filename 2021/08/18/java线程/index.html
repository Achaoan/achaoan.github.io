Java 多线程详解

1. 多线程的实现方式

四种方式：

方式一：继承 Thread 类

方式二：实现 Runnable 接口

方式三：实现 Callable 接口 + FutureTask（有返回值）

方式四：线程池（推荐）

2. 常见的线程池

  • FixedThreadPool：固定核心线程数，无界队列
  • CachedThreadPool：核心 0，最大 Integer.MAX_VALUE，SynchronousQueue
  • SingleThreadExecutor：单线程，无界队列
  • ScheduledThreadPool：支持定时和周期性任务

注意：阿里 Java 开发手册强制要求通过 ThreadPoolExecutor 手动创建，不允许使用 Executors。

3. 线程池数量设置多少？

  • CPU 密集型：线程数 = CPU 核心数 + 1
  • IO 密集型：线程数 = CPU 核心数 × 2（或 Nthreads = Ncpu × Ucpu × (1 + W/C)）

4. 线程池的参数（7大核心参数）

参数 说明
corePoolSize 核心线程数
maximumPoolSize 最大线程数
keepAliveTime 临时线程空闲存活时间
unit 时间单位
workQueue 阻塞队列
threadFactory 线程工厂
handler 拒绝策略

5. 线程池执行流程

6. 四种拒绝策略

  • AbortPolicy（默认）：抛出 RejectedExecutionException
  • CallerRunsPolicy：调用者线程自己执行
  • DiscardPolicy：直接丢弃，不抛异常
  • DiscardOldestPolicy：丢弃队列中最老的任务

7. 线程池状态

8. ThreadLocal 原理

每个线程维护一个 ThreadLocalMap，以 ThreadLocal 为 key，实际存储值为 value。常用于存储数据库连接、Session 等线程隔离数据。

9. synchronized 和 Lock 的区别

特性 synchronized Lock
关键字/接口 Java 关键字 接口（ReentrantLock）
锁释放 自动释放 手动 unlock()
锁状态 无法判断 可判断
可重入 支持 支持
公平性 非公平 可设置公平/非公平
中断响应 不支持 支持 lockInterruptibly()

10. volatile 的作用

  • 保证可见性：线程修改后立即写入主存，其他线程缓存失效
  • 禁止指令重排序：内存屏障
  • 不保证原子性

11. sleep 和 wait 的区别

  • sleep 是 Thread 方法，不释放锁；wait 是 Object 方法，释放锁
  • sleep 可在任何地方使用；wait 只能在 synchronized 块中使用
  • sleep 需要捕获 InterruptedException；wait 不需要

12. 死锁的四个必要条件

  1. 互斥：资源一次只能被一个线程占用
  2. 请求与保持：线程持有资源时请求其他资源
  3. 不可剥夺：资源不能被强制剥夺
  4. 循环等待：多个线程形成循环等待链

13. CAS（Compare And Swap）

乐观锁技术，包含三个操作数：内存地址 V、预期值 A、新值 B。当 V 的值等于 A 时，将 V 的值更新为 B。ABA 问题可通过 AtomicStampedReference 解决。

14. AQS（AbstractQueuedSynchronizer）

Java 并发包的基石，通过内置的 FIFO 队列 + volatile state 状态字段实现。ReentrantLock、CountDownLatch、Semaphore 等都是基于 AQS 实现。
