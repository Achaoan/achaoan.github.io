SpringBoot基础及相关问题

本文最后更新于：2 小时前

一、Spring Boot 简介

Spring Boot 是 Spring 框架的快速开发脚手架，核心设计理念是 **”约定优于配置”**。

主要特点

  • 自动配置：根据 classpath 下的依赖自动配置 Bean
  • 内嵌服务器：默认集成 Tomcat/Jetty/Undertow，支持 java -jar 独立运行
  • 起步依赖（Starter）：将常用依赖打包，一行引入即可
  • 生产就绪：监控（Actuator）、健康检查、外部化配置
  • 无需 XML 配置：纯 Java 注解 + 配置文件

二、启动流程

Spring Boot 启动核心流程（高频面试题）：

  1. 启动 main() 方法 → 调用 SpringApplication.run()
  2. 创建 SpringApplication 对象 → 推断应用类型、设置初始化器、监听器、确定主应用类
  3. 准备环境（Environment） → 加载 application.properties/yml、系统环境变量、命令行参数
  4. 创建并刷新 ApplicationContext → 加载配置类、自动配置类、注册 Bean、依赖注入
  5. 启动嵌入式 Web 服务器（如 Tomcat）→ 注册 Servlet、Filter
  6. 发布 ApplicationStartedEvent → 触发状态监听逻辑
  7. 执行 CommandLineRunner / ApplicationRunner → 启动后初始化逻辑
  8. 发布 ApplicationReadyEvent → 应用进入运行状态

三、自动配置原理（Auto-Configuration）

这是 Spring Boot 的灵魂机制，也是面试最高频考点。

实现原理三步走

步骤 说明
① @EnableAutoConfiguration 核心注解，通过 @Import(AutoConfigurationImportSelector.class) 导入自动配置类
② AutoConfigurationImportSelector 加载 META-INF/spring.factories（2.7+ 改为 AutoConfiguration.imports）中注册的自动配置类
③ 条件注解过滤 使用 @ConditionalOnClass、@ConditionalOnMissingBean 等判断是否生效

核心条件注解

  • @ConditionalOnClass：classpath 存在指定类时生效
  • @ConditionalOnMissingBean：容器中不存在指定 Bean 时生效
  • @ConditionalOnProperty：根据配置属性条件生效
  • @ConditionalOnWebApplication：当前是 Web 应用时生效

五、常见面试题

1. 如何理解 Starter？

Starter 是一组有用的依赖集合。引入 spring-boot-starter-web 会自动引入 Spring MVC、Tomcat、Jackson 等依赖（利用 Maven 传递依赖），无需开发者手动添加。

2. Spring Boot 默认 Tomcat 能处理多少请求？

默认核心线程数 10，最大线程数 200。Tomcat 线程池改造了 JDK 策略：核心线程满后直接创建到最大线程数（不排队），所以同一时刻最多处理 200 个请求。

3. 配置文件的加载优先级？

命令行参数 > JAR 包外部配置文件 > JAR 包内部配置文件 > application-{profile}.properties > application.properties > application.yml

4. 如何实现全局异常处理？

使用 @ControllerAdvice + @ExceptionHandler 统一处理。

5. Spring Boot 事件机制？

基于 Spring 的发布-订阅模式（观察者模式）：

  • 创建事件类（继承 ApplicationEvent）
  • 使用 ApplicationEventPublisher 发布
  • 使用 @EventListener 监听

6. 如何自定义 Starter？

  1. 创建 autoconfigure 模块，定义自动配置类（@Configuration + @Conditional）
  2. 在 META-INF/spring.factories 或 AutoConfiguration.imports 中注册
  3. 创建 starter 模块，依赖 autoconfigure
  4. 打包后其他项目引入即可自动生效

7. 如何实现异步处理？

  • 使用 @Async 注解（需在主类加 @EnableAsync）
  • 使用 CompletableFuture
  • 使用 @Scheduled 定时任务（需加 @EnableScheduling）

8. 如何实现 Session 共享？

将 Session 存储在 Redis 中，各微服务通过 Redis 读取 Session，实现共享。Spring Session 基于代理过滤器实现。
