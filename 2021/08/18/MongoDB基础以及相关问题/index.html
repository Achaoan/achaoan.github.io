MongoDB基础以及相关问题

本文最后更新于：3 小时前

一、MongoDB 简介

MongoDB 是一个基于分布式文件存储的 NoSQL 数据库，旨在为 Web 应用提供可扩展的高性能数据存储解决方案。它将数据存储为 BSON 格式的文档（类似于 JSON 对象），数据结构由键值对组成。

核心术语对照

MySQL MongoDB
数据库（Database） 数据库（Database）
表（Table） 集合（Collection）
行（Row） 文档（Document）
列（Col） 字段（Field）
主键（Primary Key） 对象 ID（ObjectId）
索引（Index） 索引（Index）

什么是 BSON？

BSON（Binary JSON）是 JSON 文档的二进制表示：

  • 遍历速度优于 JSON（有长度前缀）
  • 支持更多的数据类型（Date、Binary、ObjectId 等）
  • 存储空间比 JSON 略大

ObjectId 结构

ObjectId 共 12 字节（24 位十六进制字符串）：

  • 4 字节：时间戳（秒级）
  • 5 字节：随机值（机器标识 + 进程 ID）
  • 3 字节：自增计数器

保证分布式环境下 ID 的唯一性，无需自增。

二、优缺点

优点

  • 文档存储：数据以 BSON 文档存储，结构灵活，方便迭代开发
  • 模式自由：集合不需要定义模式，同一集合中文档结构可以不同
  • 高性能：支持多种索引，热数据常驻内存，读写速度快
  • 水平扩展：支持分片集群，自动切分数据，添加节点即可扩容
  • 高可用：支持复制集，主节点故障时自动选举新主节点
  • ACID 事务：4.0+ 支持多文档事务，4.2+ 支持分布式事务
  • 丰富查询：CRUD、聚合管道、文本搜索、地理空间查询

缺点

  • 不支持 JOIN：虽然 $lookup 可模拟，但性能不如 SQL JOIN
  • 内存占用大：热数据常驻内存，对内存要求较高
  • 成熟度相对较低：相比 MySQL/Oracle 社区较小
  • 数据冗余：反范式化设计可能导致数据冗余

三、常见面试题

1. MongoDB 与 MySQL 的区别？

维度 MySQL MongoDB
数据模型 关系型（表、行、列） 文档型（集合、文档）
Schema 固定模式 无模式（动态）
查询语言 SQL 查询 API（JSON-like）
事务 支持完善 4.0+ 支持
扩展方式 垂直扩展为主 水平扩展（分片）
适用场景 强一致性、复杂事务 大数据量、快速迭代

2. MongoDB 支持哪些索引类型？

  • 单字段索引、复合索引、多键索引（数组字段）
  • 哈希索引（精确匹配）、文本索引（全文搜索）
  • 地理空间索引（2dsphere、2d）、TTL 索引（过期自动删除）
  • 唯一索引

3. 什么是复制集（Replica Set）？

一组维护相同数据集的 mongod 实例：

  • 1 个主节点（Primary）：处理写入操作
  • N 个从节点（Secondary）：复制主节点数据，可分担读请求
  • 自动故障转移：主节点故障时自动选举新主节点

4. 什么是分片（Sharding）？

将数据分布在多个 MongoDB 实例上：

  • 分片键（Shard Key）：决定数据分配到哪个分片
  • 组件：mongos（路由器）、config server（配置）、shard（数据分片）
  • 自动均衡：数据在各分片间均匀分布

5. 聚合管道是什么？

一系列数据处理阶段，每个阶段对文档进行转换后输出到下一阶段：

常用操作符：$match、$project、$group、$sort、$lookup、$unwind

6. MongoDB 适用场景？

  • 内容管理（博客、CMS、评论系统）
  • 日志存储、实时分析、物联网
  • 社交应用（朋友圈、地理位置查询）
  • 电商商品目录（非结构化数据）
  • 缓存层（配合 MySQL 使用）

不适合：复杂事务、多表关联查询、强一致性要求极高的金融系统。
