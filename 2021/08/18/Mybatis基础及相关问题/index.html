Mybatis基础及相关问题

本文最后更新于：3 小时前

一、MyBatis 简介

MyBatis 是一款基于 Java 的半自动化 ORM（对象关系映射）持久层框架，前身是 iBatis。它内部封装了 JDBC，使开发者只需要关注 SQL 语句本身。

核心优势：

  • 半自动化 ORM：SQL 需手动编写，灵活可控
  • 动态 SQL：通过标签灵活拼接 SQL
  • 缓存机制：内置一级缓存和二级缓存
  • 易于整合：无缝集成 Spring、Spring Boot

与 Hibernate 的区别：

对比维度 MyBatis Hibernate
ORM 类型 半自动（SQL 手动编写） 全自动（SQL 自动生成）
SQL 控制度 高 低
学习成本 低（专注 SQL 映射） 高（需掌握 HQL、缓存等）
适用场景 复杂 SQL、互联网项目 简单 CRUD、企业级应用

二、核心组件

组件 作用
SqlSessionFactoryBuilder 读取配置文件创建 SqlSessionFactory
SqlSessionFactory 创建 SqlSession，应用级单例
SqlSession 会话对象，提供 CRUD 操作 API
Mapper 接口 无需实现类，MyBatis 通过动态代理生成
Mapper.xml SQL 映射文件
Executor 核心执行引擎

三、工作原理

  1. 加载配置（mybatis-config.xml + Mapper.xml）→ Configuration 对象
  2. 构建 SqlSessionFactory
  3. 创建 SqlSession
  4. getMapper() 通过 JDK 动态代理生成 Mapper 接口的代理对象
  5. 代理对象触发 Executor 执行 SQL
  6. Executor 通过 StatementHandler 处理 JDBC 操作
  7. ParameterHandler 绑定参数
  8. ResultSetHandler 将结果集映射为 Java 对象
  9. 关闭 SqlSession

五、常见面试题

1. #{} 和 ${} 的区别？

对比 #{} ${}
本质 预编译参数占位符（?） 字符串拼接
安全性 防 SQL 注入 存在 SQL 注入风险
适用场景 传递参数 动态表名、列名

结论：优先用 #{}，仅动态表名/列名时用 ${}。

2. resultType 和 resultMap 区别？

  • resultType：直接映射到 Java 对象（字段名需与列名一致）
  • resultMap：通过 <resultMap> 标签定义字段映射关系，更灵活

5. Mapper 接口为什么不需要实现类？

MyBatis 使用 JDK 动态代理 为 Mapper 接口生成代理对象：

  • 接口全限定名 → namespace
  • 接口方法名 → SQL 的 id
  • 方法参数 → SQL 参数

注意：Mapper 接口中的方法不能重载。

6. MyBatis 的一级缓存和二级缓存？

  • 一级缓存（SqlSession 级别）：默认开启，同一个 SqlSession 内重复查询直接返回缓存
  • 二级缓存（Mapper 级别）：默认关闭，需手动配置 <cache>，跨 SqlSession 共享
  • 查询顺序：先查二级缓存 → 再查一级缓存 → 最后查数据库

7. 分页如何实现？

  • 逻辑分页：RowBounds（不推荐，全部查出后内存分页）
  • 物理分页：手动写 LIMIT，或使用 PageHelper 插件（原理：通过 MyBatis 插件机制拦截 SQL，重写添加分页语句）

8. MyBatis 涉及的设计模式？

  • 建造者模式（SqlSessionFactoryBuilder）
  • 工厂模式（SqlSessionFactory）
  • 代理模式（Mapper 接口、延迟加载）
  • 模板方法模式（Executor）
  • 责任链模式（插件拦截器）
