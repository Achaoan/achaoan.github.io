上下文窗口与长文本处理：RoPE、长上下文技术演进

本文最后更新于：1 分钟前

一、什么是上下文窗口

上下文窗口（Context Window）是 LLM 一次能”看到”的最大 Token 数量。超出窗口的内容会被截断，模型无法感知。

关键数据点：

  • GPT-3 (2020)：4K Token
  • GPT-4 (2023)：8K → 32K → 128K
  • Claude 3 (2024)：200K
  • Gemini 1.5 Pro (2024)：2M Token
  • GPT-4o (2025)：128K

主流模型已从几千 Token 增长到数十万乃至百万级别。但长上下文并不意味着长上下文有效——模型在窗口中间段的”迷失”是另一个问题。

二、位置编码的演进

Transformer 的 Attention 是置换不变的——如果没有位置信息，”我打你”和”你打我”的 Attention 计算完全相同。位置编码就是给模型注入 Token 的位置信息。

绝对位置编码（原始 Transformer）

每个位置 i 对应一个固定的正弦余弦位置向量。

局限：

  • 训练时固定最大长度（超出无法处理）
  • 位置向量与 Token 向量相加，可能干扰语义
  • 无法外推到更长的序列

RoPE（Rotary Position Embedding）

RoPE 是目前主流模型（GPT-4、Llama、Mistral、Gemma）使用的方案。它通过旋转矩阵将位置信息注入 Q 和 K 向量。

外推（Extrapolation）： 模型训练时只在 8K 窗口上训练，但推理时可以用到 32K——这就是 RoPE 的外推能力。实际效果会逐渐衰减，但比绝对位置编码好得多。

ALiBi（Attention with Linear Biases）

在 Attention 得分上加一个与距离成正比的偏置（惩罚），越远的 Token 偏置越大。

优势： 训练时只需短序列，推理时可直接外推超长序列——因为偏置只依赖距离，不依赖绝对位置。Bloomberg GPT 和某些 MosaicML 模型使用。

三、长上下文技术演进

窗口缩放（Scaling）

最简单的方案——直接扩大上下文窗口。代价是 Attention 的计算量 O(N²) 和 KV Cache 占用 O(N) 都增长。

YaRN（Yet another RoPE extensioN）

改进 RoPE 外推的方法，结合插值和 NTK-aware 缩放，在 Llama 等模型上实现 2-4 倍窗口扩展，无需额外微调。

Ring Attention / 分布式注意力

将超长序列切分到多个 GPU 上，每个 GPU 负责一段序列的 Attention 计算，通过通信传递 KV。理论上可以扩展到任意长度——只要 GPU 够多。

四、”迷失在中间”问题

即使模型有 128K 窗口，位于窗口中间的信息最容易被模型忽略。

Lost in the Middle（Liu et al., 2023）的研究发现：

  • 开头和结尾的文档召回率 > 90%
  • 中间文档的召回率 < 50%
  • 这是一个位置偏差——模型天然倾向关注开头和结尾

五、Streaming LLM 与无限上下文

Streaming LLM（2023）提出了一种方案：在生成过程中只保留开头 Token 和最近的 Token，丢弃中间部分，让模型理论上可以无限生成。

Attention Sink 现象： 模型会把大量注意力分配给第一个 Token（SOS），即使它与当前内容无关。保留第一个 Token 能稳定注意力分布。

七、面试题

1. RoPE 位置编码的原理和优点？

RoPE 通过旋转矩阵将位置信息编码到 Q 和 K 向量中，使注意力只依赖于相对位置差。优点是可外推（训练短窗口、推理长窗口）、与 Attention 自然融合、有远程衰减。

2. 什么是”Lost in the Middle”问题？

长上下文模型中，处于窗口中间的信息召回率显著低于开头和结尾。原因是模型的位置偏差导致中间段注意力不足。缓解方法：关键信息放两端、控制检索数量。

3. 长上下文和 RAG 的关系？

两者是互补方案。长上下文允许一次性放入大量信息但存在”迷失在中间”问题和高计算成本。RAG 有选择性地只检索相关文档效率更高。实际系统通常结合：RAG 筛选出关键文档 + 放入有限的上下文窗口。

4. 上下文窗口从 4K 扩展到 128K 面临哪些挑战？

计算量 O(N²) 爆炸、KV Cache 显存 O(N) 增长、注意力分布稀疏化（很多注意力分数趋近于 0）、模型在中间段丢失焦点。解决方案包括 RoPE 外推、Flash Attention、GQA/MQA 减少 KV 缓存。
