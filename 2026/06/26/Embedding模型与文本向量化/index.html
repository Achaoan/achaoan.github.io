Embedding 模型与文本向量化：原理、相似度与应用

本文最后更新于：3 分钟前

一、什么是 Embedding

Embedding（嵌入）是将文本、图片等非结构化数据映射到高维向量空间的技术。在这个向量空间中，语义相近的文本距离更近，语义无关的文本距离更远。

Embedding 是 RAG、语义搜索、聚类分类等应用的底层技术。

二、文本转向量的原理

从词向量到句向量

Word2Vec（里程碑）：

  • 通过 CBOW（上下文预测中心词）或 Skip-gram（中心词预测上下文）训练
  • 每个词对应一个固定向量
  • 局限：无法处理一词多义

BERT Embedding：

  • 利用双向 Transformer 编码器
  • 同一词在不同上下文中有不同的向量
  • 取 [CLS] Token 的输出或所有 Token 的平均池化作为句向量

OpenAI Embedding 模型：

  • text-embedding-3-small：1536 维，速度快
  • text-embedding-3-large：3072 维，精度高，可降维
  • 专为语义搜索优化

三、相似度计算方法

方法 公式 范围 特点
余弦相似度（最常用） cos(θ) = (A·B) / (‖A‖·‖B‖) [-1, 1] 只考虑方向，不考虑长度
欧氏距离 d = ‖A - B‖₂ [0, ∞) 考虑方向和长度
点积 A·B (-∞, ∞) 需要向量已归一化
曼哈顿距离 Σ aᵢ - bᵢ

余弦相似度是 Embedding 搜索的默认选择，因为它对向量长度不敏感——在 Embedding 空间中，向量的方向比长度更能反映语义。

五、Embedding 在 RAG 中的角色

RAG 的核心流程中，Embedding 出现两次：

关键指标：

  • 召回率（Recall）：前 K 个结果包含相关文档的比例
  • 延迟：< 100ms 为优（取决于向量维度和文档数量）
  • 存储：100 万条 1024 维向量约占用 4GB（含索引）

影响搜索质量的因素：

  • Embedding 模型与领域语言的匹配度
  • 分块策略（chunk size / overlap）
  • 索引算法（HNSW / IVF / Flat）
  • 是否用重排序（Reranker）二次过滤

七、面试题

1. Embedding 余弦相似度的计算过程？

向量 A 和 B 的点积除以各自模长的乘积。值越接近 1 表示语义越相似，越接近 -1 表示语义越相反。余弦相似度只关注方向，不关注向量长度。

2. Embedding 模型和 LLM 的区别？

Embedding 模型输出高维向量（语义表示），LLM 输出 Token 概率（文本生成）。Embedding 模型通常更小更快，用于检索和分类；LLM 用于理解和生成文本。

3. 怎么评估 Embedding 质量？

用语义相似度基准（如 STS-B / MTEB）评估，或在具体任务（RAG 召回率、分类准确率）上评测。粗糙做法：人工检查 Top-K 结果的相关性。

4. text-embedding-3-large 输出 3072 维，为什么可以降维？

它使用 Matryoshka 表示学习，同一个模型可以输出 256/512/1024/2048/3072 维。降维时去掉末尾维度，精度略有下降但存储和检索效率提升。选型时根据精度-效率权衡决定。
