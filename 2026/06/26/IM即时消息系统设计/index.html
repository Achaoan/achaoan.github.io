六、面试题

1. IM 系统的核心挑战？

海量长连接管理（千万级并发）、消息可靠性（不丢不重不乱序）、集群通信（跨服务器路由）、离线消息同步。

2. WebSocket 集群怎么路由消息？

Redis 存储用户→服务器映射，发送方服务器查表找到接收方服务器，通过内部 RPC 转发。MQ 消息总线作为备选方案。

3. 怎么保证消息不丢不重？

全局消息 ID 去重、发送端重试+服务端幂等、ACK 确认机制。消息先写 DB 再推送，推送失败可恢复。

4. 离线消息怎么处理？

离线消息表记录消息 ID，用户上线后批量拉取。控制上限（1000 条/7 天），超限去历史表查询。群聊离线消息合并通知。
