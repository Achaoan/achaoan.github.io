注意力机制（Attention）详解：Scaled Dot-Product、Multi-Head、KV Cache

本文最后更新于：2 分钟前

一、什么是注意力机制

注意力（Attention）是 Transformer 架构的核心。简而言之，注意力机制让模型在处理每个 Token 时，关注序列中所有其他 Token，并根据相关性加权组合它们的信息。

三、Multi-Head Attention

单一注意力只能捕获一种位置关系。Multi-Head 将 Q、K、V 拆分成多个子空间，每个 Head 学习不同的关注模式。

GPT-4 的注意力配置（推测）：

  • 层数：~120
  • Head 数：~64
  • Head 维度：~128
  • 总维度：8192

每个 Head 在不同层关注不同事物。底层更多关注语法和词汇关系，高层关注语义和长距离依赖。

五、Causal Mask（因果掩码）

解码器中的 Self-Attention 需要防止”看到未来的 Token”——这就是 Causal Mask（也称为 Autoregressive Mask / 下三角掩码）。

这是 GPT / Claude 等生成式模型的核心设计——训练时预测下一个 Token 必须基于已生成的内容。

六、KV Cache

KV Cache 是 LLM 推理加速最核心的优化手段。每次生成一个 Token 时，前半部分的 K、V 矩阵不需要重新计算。

  • PagedAttention（vLLM）：按页管理缓存，减少碎片

七、Attention 的优化变体

Flash Attention

效果： 相同 GPU 能处理 2-4 倍更长的序列，且计算速度不降反升。

稀疏 Attention

部分注意力权重可以跳过（某些位置的相关性很低），减少计算量。

Linear Attention

用核技巧（kernel trick）将 Softmax Attention 的计算复杂度从 O(N²) 降到 O(N)，适合超长序列。

八、面试题

1. Attention 公式中各部分的作用？

Q（查询）提出”找什么”，K（键）表示”有什么”，V（值）提供”是什么”。Q×K^T 计算相似度，√d_k 缩放防止梯度消失，softmax 归一化权重，×V 加权组合信息。

2. Multi-Head Attention 为什么有效？

单头只能捕获一种关系模式，多头让模型在不同子空间中学习不同类型的依赖（语法、语义、位置、指代），最后拼接融合。每个 Head 关注不同的”视角”。

3. KV Cache 的原理是什么？

生成过程中，前面 Token 的 K 和 V 矩阵在不同解码步之间不变，因此缓存复用。每次生成新 Token 只需计算当前 Token 的 K、V，避免重复计算。代价是显存随序列长度线性增长。

4. Causal Mask 的作用？

解码器中，每个 Token 只能看到自己和左边的 Token，不能”偷看”未来 Token。通过上三角掩码（设为 -∞）实现。这是自回归模型生成的核心约束。

5. Flash Attention 解决了什么问题？

标准 Attention 的 N×N 注意力矩阵占用 O(N²) 显存，长序列时 GPU 显存不够。Flash Attention 通过分块计算和重计算，将显存降到 O(N)，同时利用 SRAM 加速，能处理 2-4 倍更长的序列。
