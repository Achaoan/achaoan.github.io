七、面试题

1. 布隆过滤器的原理？

使用位数组 + 多个哈希函数。添加时将 k 个位置置为 1，查询时检查 k 个位置。全部为 1 表示可能存在，有 0 表示一定不存在。

2. 布隆过滤器的误判率怎么控制？

通过 m/n（位数组大小/元素数量）和 k（哈希函数个数）控制。Guava 中直接传入预期元素数量和可接受误判率即可。

3. 布隆过滤器能删除吗？

普通布隆过滤器不支持删除（哈希冲突无法分辨）。Counting Bloom Filter 用计数器替代位数组，支持删除。

4. 如何用布隆过滤器解决缓存穿透？

预热时把所有合法 key 加载到布隆过滤器，查询时先检查布隆过滤器，不存在的 key 直接返回，不查数据库。

5. Redis 布隆过滤器怎么用？

Redis Stack 支持 BF.ADD/BF.EXISTS 命令。也可以基于 Redis BitMap 自行实现。
