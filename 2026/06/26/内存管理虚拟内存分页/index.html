四、面试题

1. 虚拟内存的作用？

隔离地址空间（进程间互不干扰）、部分加载（程序不需要全部装入内存）、连续地址空间（屏蔽物理内存碎片）、共享内存（共享库只存一份）。

2. 多级页表解决的问题？

节省内存。32 位单级页表要 4MB（每个进程），多级页表按需分配，只保存进程实际用到的地址范围的页表项。

3. 缺页中断的流程？

CPU 访问虚拟地址 → 页表中无有效映射 → 触发缺页中断 → OS 处理：内存有空闲则从磁盘加载，已满则页面置换算法选出牺牲页（脏页写回磁盘）→ 加载新页 → 更新页表 → 重新执行指令。

4. LRU 和 Clock 算法的区别？

LRU 淘汰最久未用的页，精度高但实现成本高。Clock（时钟算法）是 LRU 的近似实现，用循环链表 + 访问位扫描，开销小，Linux 实际使用。

5. TLB 是什么？

页表的 CPU 缓存，加速虚拟地址到物理地址的转换。TLB 命中时不需要查内存中的页表，大大降低地址转换开销。进程切换需要刷新 TLB。
