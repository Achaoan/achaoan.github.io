分词器（Tokenizer）详解：BPE、WordPiece、Token 与成本

本文最后更新于：3 分钟前

一、什么是 Tokenizer

Tokenizer（分词器）是 LLM 的第一道关卡——将原始文本转换为模型能理解的 Token 序列。用户输入的每一句话，都要经过分词器拆成 Token，再交给模型。模型的输出同样要从 Token 序列解码回文本。

Token 是 LLM 计费、上下文窗口、性能的核心单位。理解分词器对高效使用 AI API 至关重要。

二、主流分词算法

BPE（Byte Pair Encoding）

BPE 是最广泛使用的分词算法，GPT、Claude、Llama 等主流模型都使用它或其变体。BPE 从单字符级别开始，逐步合并最常出现的相邻 Token 对。

训练步骤：

  1. 将语料拆成单字符序列（含词尾标记）
  2. 统计所有相邻 Token 对的频率
  3. 合并频率最高的 Token 对为新 Token
  4. 重复 2-3 步，直到词表达到目标大小

特点：

  • 数据驱动：合并频率基于训练语料统计
  • 词表大小可控（典型值 32K-128K）
  • 任意输入都能分词（不会出现 UNK）

WordPiece

WordPiece 由 Google 提出，用于 BERT 系列模型。与 BPE 的区别在于合并标准：BPE 按频率合并，WordPiece 按似然增益（互信息）合并。

特点：

  • 更注重语义组合性
  • BERT 系列模型使用
  • 对罕见词拆分为子词

SentencePiece

Google 推出的分词框架，将字节序列视为统一编码，不预先假设语言边界。支持 BPE 和 Unigram 两种算法。

关键区别： 传统分词器需要先做空格分词（英文天然优势），SentencePiece 将整个句子作为原始输入，对中文等连续文字更友好。

Unigram 算法的特点：

  • 基于概率模型
  • 每次移除”最不重要”的 Token（按损失期望）
  • 可根据需要生成不同粒度的分词结果

算法对比

特性 BPE WordPiece SentencePiece (Unigram)
代表模型 GPT, Claude, Llama BERT T5, Gemma, ALBERT

三、特殊 Token

每个 Tokenizer 都包含一组特殊 Token，它们不表示具体文本内容，而是携带结构信息。

不同模型的处理方式不同：

  • GPT/Claude 不使用 [SEP]、[CLS]，依赖格式换行和角色标记（如 <|im_start|>）
  • BERT 依赖 [CLS] 和 [SEP] 完成分类和句对输入
  • Llama 使用 <s> 和 </s>

四、中文分词的特殊性

中文没有空格分隔，分词方式与英文有本质区别。

字节级方案（主流趋势）：

  • GPT-4、Claude 3+ 等在字节级别运行 BPE
  • 一个中文字通常编码为 1-2 个 Token
  • 不依赖语言的预分词规则
  • 统一处理所有语言和特殊字符

Token 数量对比（GPT-4 分词器）：

内容 字符数 Token 数 Token/字符
“Hello, world!” 13 3 0.23
“我爱编程” 4 2 0.50
英文新闻 1000字 1000 ~250 0.25
中文新闻 1000字 1000 ~600 0.60

实践意义： 中文 Token 消耗是英文的 2-3 倍，API 调用成本也相应增加。

五、Token 对成本与性能的影响

影响 Token 数量的因素：

  • 语言：中文 Token 密度更高
  • 格式：JSON 和代码含大量空格换行
  • 特殊符号：每换行、缩进都消耗 Token
  • 重复内容：每个重复片段都重新计 Token

降低 Token 消耗的技巧：

  • System Prompt 精简，去除非必要指令
  • Few-shot 示例挑选最精炼的
  • 使用 max_tokens 限制输出长度
  • 定期清理历史消息（滑动窗口）
  • 结构化数据用压缩格式（如 TSV 替代 JSON）

七、Tokenizer 与模型的绑定关系

每个 Tokenizer 对应一个固定的词表（vocab.json），一旦模型训练完成，Token 映射关系就冻结了。这意味着：

  • 同一个文本用不同模型的 Tokenizer 分出的 Token 数不同
  • 迁移模型时必须切换对应 Tokenizer
  • Tokenizer 的优化只能在模型预训练之前发生
  • 特殊 Token 的处理方式随模型架构固定

关键提醒： 不能通过优化 Tokenizer 来降低推理成本。Token 消耗不是用户能优化的——同一段文本的 Token 数由训练时固化的 Tokenizer 决定。减少 Token 的核心方式是精简输入。

八、面试题

1. BPE 和 WordPiece 的区别是什么？

合并标准不同：BPE 按频率合并最常出现的 Token 对，WordPiece 按互信息（似然增益）合并。BPE 代表模型 GPT/Claude/Llama，WordPiece 代表模型 BERT。

2. 为什么中文比英文消耗更多 Token？

中文没有空格分隔，字节级 BPE 将每个汉字编码成独立或组合 Token。相同字符数下，中文 Token 密度更高（约 0.6 Token/字 vs 英文 0.25 Token/字）。

3. Tokenizer 在模型中的作用是什么？

Tokenizer 是 LLM 的第一层，将原始文本映射为 Token ID 序列。模型只能处理 Token ID，不能直接处理字符。Tokenizer 的性能决定了模型能”看到”什么粒度的信息。

4. 如何估算 Token 数？

英文约 4 字符/Token，中文约 1.5-2 字符/Token。更精确用 tiktoken（OpenAI）或对应模型的 Tokenizer 库。API 调用时通常响应头会返回 Token 消耗数。
