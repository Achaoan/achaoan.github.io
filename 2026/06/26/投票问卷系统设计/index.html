一、投票/问卷系统的核心功能

投票和问卷系统允许创建者发布问题，用户选择答案。核心挑战在高并发写入和实时计数。

六、面试题

1. 高并发投票系统的设计思路？

Redis 计数 + 异步落库。Redis INCR 实时计数，MQ 缓冲异步写入 MySQL。查询直接走 Redis 毫秒级响应，MySQL 只做持久化。

2. 怎么防止刷票？

多层防线：登录验证、Redis + MySQL 双重去重、频率限制（IP/用户粒度）、风控策略（异常行为检测）。每层拦截不同的攻击手段。

3. Redis 和 MySQL 数据不一致怎么办？

接受最终一致性。投票场景允许秒级延迟，Redis 计数为主，MySQL 异步落库。Redis 宕机时从 MySQL COUNT 恢复计数。

4. 投票结束如何结算？

截止时间到后，开启状态为已结束。从 Redis 同步最终结果到 MySQL 结果表，清理 Redis 缓存。客户端不再显示投票按钮，只展示结果。
