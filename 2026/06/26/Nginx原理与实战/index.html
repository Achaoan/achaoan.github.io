1. Nginx 为什么性能高？

事件驱动 + 异步非阻塞模型（epoll）。单 Worker 可以处理大量连接，不用为每个连接创建线程。相比 Apache 的进程/线程模型，内存和上下文切换成本极低。

2. 反向代理和正向代理的区别？

正向代理代理客户端（翻墙、公司上网），隐藏客户端。反向代理代理服务端（负载均衡），隐藏后端服务器。

3. Nginx 负载均衡策略？

轮询（默认）、weight（权重）、ip_hash（IP 绑定）、least_conn（最少连接）、fair（响应时间）、url_hash（URL 哈希）。

4. 如何用 Nginx 做限流？

limit_req 限制请求频率（rate + burst），limit_conn 限制并发连接数，limit_rate 限制带宽。

5. location 匹配优先级？

= 精确 > ^~ 前缀 > ~ 正则 > 普通前缀。

6. Nginx 如何处理 HTTP 请求？

接收到请求 → 解析请求行/头 → 匹配 location → 执行模块（静态文件/反向代理/rewrite 等）→ 返回响应。
