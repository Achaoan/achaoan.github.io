1. OAuth2 的授权码流程？

用户点击授权 → 跳转到授权服务器 → 用户同意 → 返回授权码 → 后端用授权码换 Token → 用 Token 请求资源。

2. JWT 和 Session 的区别？

JWT 无状态（客户端存储），适合分布式系统。Session 有状态（服务端存储），注销方便但需要集中存储。JWT 用签名防篡改，Payload 不加密。

3. SSO 单点登录的原理？

统一认证中心。系统 A 验证用户 → 认证中心生成 Token → 用户访问系统 B → 认证中心确认已登录 → 直接放行。核心是共享认证 Session。

4. Refresh Token 和 Access Token 的区别？

Access Token 短期（15 分钟），用于请求资源。Refresh Token 长期（7 天），用于获取新的 Access Token。避免长期暴露 Access Token 降低风险。

5. PKCE 是什么？

Proof Key for Code Exchange，在授权码基础上增加 code_challenge 和 code_verifier，防止授权码被截获后使用。OAuth2.1 强制要求。
