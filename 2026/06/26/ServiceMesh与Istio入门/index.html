九、面试题

1. Service Mesh 是什么？解决了什么问题？

将服务通信能力（服务发现、负载均衡、熔断、限流）从业务代码中剥离到基础设施层，通过 Sidecar 代理实现。解决微服务框架的 SDK 耦合、升级困难、多语言支持差等问题。

2. Istio 的架构组件？

数据平面（Envoy Sidecar）处理流量，控制平面（Istiod）包含 Pilot（流量管理）、Citadel（安全）、Galley（配置管理）。

3. Sidecar 的工作原理？

每个 Pod 中与业务容器并列运行 Envoy 代理，通过 iptables 劫持所有进出流量，实现对通信的透明管控。

4. Istio 如何实现灰度发布？

通过 VirtualService 定义路由规则，按权重或请求头将流量分配到不同版本子集（subset）。比例可精确到 1%。

5. Istio 和 Spring Cloud 怎么选？

Istio 无侵入、多语言友好，适合云原生和异构系统。Spring Cloud 学习成本低、调试容易，适合纯 Java 中小团队。两者可以共存使用。
