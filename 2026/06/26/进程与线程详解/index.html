进程与线程详解：概念、状态、IPC 与线程模型

本文最后更新于：8 分钟前

一、进程与线程的概念

进程（Process）

进程是资源分配的最小单位，是程序的一次执行过程。

线程（Thread）

线程是CPU 调度的最小单位，是进程内的一个执行流。

进程 vs 线程

对比维度 进程 线程
地址空间 独立 共享进程地址空间
资源开销 大（创建/切换） 小（创建/切换）
通信方式 IPC（管道、共享内存等） 直接读写变量
独立性 高（一个进程崩溃不影响其他） 低（一个线程崩溃可能影响整个进程）
上下文切换 慢（页表切换、TLB 刷新） 快（只切换寄存器）
创建速度 慢（分配独立资源） 快（复用进程资源）

四、线程模型

用户线程（ULT - User Level Thread）

内核线程（KLT - Kernel Level Thread）

轻量级进程（LWP - Light Weight Process）

三种映射模型

模型 描述 优势 劣势
1:1 一个用户线程对应一个内核线程 利用多核、阻塞不影响其他 线程创建/切换开销大
N:1 N 个用户线程对应一个内核线程 切换快、创建轻量 无法利用多核、单线程阻塞全进程
M:N M 个用户线程对应 N 个内核线程 兼顾性能和并行性 实现复杂

五、进程间通信（IPC）

方式 说明 特点 适用场景
管道（Pipe） 单向流，父子进程使用 简单、内核缓冲 Shell 管道命令
命名管道（FIFO） 文件系统路径 任意进程可通信 无亲缘关系进程
消息队列 消息链表 有格式、异步 短消息通信
共享内存 直接读写同一块内存 最快、需同步 大量数据交换
信号量 计数器 同步而非数据传输 进程同步互斥
Socket 网络通信 跨机器 分布式系统
信号（Signal） 异步通知 简单、信息有限 Ctrl+C、kill

六、面试题

1. 进程和线程的区别？

进程是资源分配最小单位，独立地址空间，切换开销大。线程是 CPU 调度最小单位，共享进程地址空间，切换开销小。进程间需要 IPC 通信，线程间可直接读写共享变量。

2. 线程切换比进程切换快在哪里？

线程共享地址空间，切换时不需要切换页表和刷新 TLB，只需要保存/恢复寄存器。进程切换需要切换地址空间、刷新 TLB、更新页表。

3. Java 线程状态有哪些？

NEW、RUNNABLE（就绪+运行）、BLOCKED（synchronized 阻塞）、WAITING（无限等待）、TIMED_WAITING（超时等待）、TERMINATED。

4. 进程间通信方式有哪些？

管道（父子进程）、命名管道（任意进程）、消息队列（短消息）、共享内存（大量数据，最快）、信号量（同步）、Socket（跨网络）、信号（异步通知）。
