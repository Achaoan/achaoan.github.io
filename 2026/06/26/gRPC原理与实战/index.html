八、面试题

1. gRPC 和 REST 的区别？

gRPC 使用 HTTP/2 + ProtoBuf 二进制协议，性能高、支持流式通信。REST 使用 HTTP/1.1 + JSON，调试友好、浏览器兼容好。gRPC 适合内部服务通信，REST 适合开放 API。

2. ProtoBuf 的编码原理？

使用 Varint（变长整数）和 ZigZag 编码。Tag-Value 结构：每个字段由 field_number + wire_type 标识，数值小的字段占用字节少。比 JSON 省 3-5 倍空间。

3. gRPC 的四种通信模式？

一元（Unary）、服务端流（Server Streaming）、客户端流（Client Streaming）、双向流（Bidirectional Streaming）。

4. HTTP/2 如何提升 gRPC 性能？

多路复用（一个 TCP 连接并行多个请求）、头部压缩（HPACK）、二进制帧、服务端推送。避免了 HTTP/1.1 的队头阻塞问题。
