一、Sentinel 概述

是什么？

Sentinel 是阿里巴巴开源的流量防卫兵，用于微服务的限流、熔断、降级。

为什么选择 Sentinel？（vs Hystrix）

对比 Sentinel Hystrix
隔离方式 信号量（不占线程） 线程池隔离（占线程）
限流方式 QPS/线程数/热点 基于信号量
熔断策略 慢调用/异常比/异常数 异常比
动态规则 推/拉模式都支持 需自行实现
控制台 完整可视化 UI 无
配置中心 Nacos/ZK/Apollo 无
维护状态 活跃维护 已停更

十、面试题

1. Sentinel 和 Hystrix 的区别？

Sentinel 信号量隔离（不占线程），Hystrix 线程池隔离（占用线程）。Sentinel 有控制台、支持热点限流、动态规则，Hystrix 已停更。

2. Sentinel 的熔断策略？

慢调用比例（RT > 阈值）、异常比例、异常数。三种策略触发后进入 Open 状态，timeWindow 后进入 Half-Open。

3. Sentinel 的限流模式？

直接拒绝、Warm Up（冷启动）、排队等待（匀速通过）、Warm Up + 排队。

4. Sentinel 如何实现实时统计？

滑动窗口机制。将时间划分为多个小窗口，每个窗口独立计数，随时间的推移窗口滑动。

5. Sentinel 资源注解 @SentinelResource 的属性？

value（资源名）、blockHandler（限流处理）、fallback（异常处理）、blockHandlerClass、fallbackClass。
