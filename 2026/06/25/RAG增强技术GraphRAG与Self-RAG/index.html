三、Self-RAG（自反思 RAG）

是什么

Self-RAG 让模型在生成过程中自我评估检索结果和自己的输出，决定是否需要检索、是否采纳检索结果。

是什么

六、HyDE（假设文档嵌入）

HyDE（Hypothetical Document Embedding）是一种检索增强技巧：先让 LLM 根据问题生成一个”假设的回答”，用这个回答的向量去检索相关文档。

七、各种增强技术的对比

八、面试题

1. GraphRAG 和传统 RAG 的区别？

GraphRAG 用知识图谱替代向量检索，擅长全局性和多跳推理问题。传统 RAG 适合局部单点查询。GraphRAG 的代价是需要先用 LLM 构建知识图谱。

2. Self-RAG 的核心机制？

Self-RAG 通过反思 Token 让模型自我评估：是否需要检索、检索结果是否相关、生成内容是否有依据。每一步都有反思，减少不必要的检索和幻觉。

3. CRAG 和 Self-RAG 的区别？

Self-RAG 评估后被动处理（忽略或使用），CRAG 评估后主动修正（改写查询重试）。CRAG 适合”必须给出回答”的场景，更适合客服场景。

4. Reranking 的作用？

在检索后、生成前，用更精确的模型对结果二次排序，把最相关的内容放到最前面。解决”检索排序不准”和”Lost in the Middle”问题。典型方案：Embedding 粗排 → Reranker 精排。

5. HyDE 的原理？

让 LLM 根据问题生成一段假设的回答，用假设回答的向量去检索。解决”问题的向量和文档的向量不在同一语义空间”的问题，本质是一种查询扩展技术。
