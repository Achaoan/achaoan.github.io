1. 秒杀系统怎么设计？

前端静态化 + 验证码，Nginx 限流，Redis 预减库存，MQ 异步下单，数据库乐观锁扣库存。

2. 怎么防止超卖？

Redis 原子减库存 + 数据库乐观锁（UPDATE ... WHERE stock > 0）。

3. 怎么处理库存预热？

秒杀前把库存加载到 Redis，请求先经过 Redis 预减，通过后再走 MQ 异步下单。

4. 秒杀接口怎么防刷？

隐藏接口地址（path 动态化）、验证码、接口限流、限购检查、黑名单。

5. 为什么不直接扣数据库库存？

数据库连接数有限，QPS 上限低。Redis + MQ 做缓冲，削峰填谷。
