JUC并发工具类详解：CountDownLatch、CyclicBarrier、Semaphore、CompletableFuture

本文最后更新于：14 分钟前

一、CountDownLatch（倒计数器）

允许一个或多个线程等待其他线程完成操作后再继续执行。

方法 说明
countDown() 计数器减 1，调用线程不会阻塞
await() 等待计数器变为 0，阻塞当前线程
await( timeout, unit) 带超时的等待

特点

  • 计数器不可重置（一次性）
  • 适合主线程等待子线程的单向同步

示例：6 个同学离开教室，班长锁门

二、CyclicBarrier（循环栅栏）

一组线程相互等待，直到所有线程都到达屏障点，然后同时继续执行。

特点

  • 可重置循环使用
  • 支持屏障动作（到达屏障后的回调）

示例：集齐 7 颗龙珠召唤神龙

CountDownLatch vs CyclicBarrier

对比 CountDownLatch CyclicBarrier
计数器 递减，不可重置 递增（到达数），可重置
方向 单向：主等子 双向：互相等待
实现 基于 AQS 共享模式 基于 Condition

三、Semaphore（信号量）

控制同时访问特定资源的线程数量。常用于限流。

方法 说明
acquire() 获取一个许可，无可用则阻塞
release() 释放一个许可
availablePermits() 当前可用许可数

示例：6 辆车抢 3 个车位

四、CompletableFuture（异步编程利器）

Future 的痛点

缺点 说明
不支持手动完成 无法主动通知结果
不支持非阻塞回调 get() 会阻塞
不支持链式调用 无法串联异步任务
不支持异常处理 无异常回调机制

核心方法

创建任务：

  • runAsync(Runnable)：无返回值异步任务
  • supplyAsync(Supplier)：有返回值异步任务

回调处理：
| 方法 | 作用 |
|——|——|
| thenApply(Function) | 对结果进行转换 |
| thenAccept(Consumer) | 消费结果，无返回值 |
| whenComplete(BiConsumer) | 结果或异常回调 |
| exceptionally(Function) | 异常处理 |
| thenCombine(CompletionStage, BiFunction) | 合并两个异步结果 |

示例：带异常处理的异步调用

电商比价案例（异步并行查询）

五、四大工具类对比

工具 核心作用 可重置 经典场景
CountDownLatch 等待多个线程完成任务 ❌ 主线程等待子线程
CyclicBarrier 线程互相等到屏障点 ✅ 多阶段并行计算
Semaphore 控制并发访问数 ✅ 限流、连接池
CompletableFuture 异步编程、回调编排 — 异步任务链、并行查询
