六、面试题

1. AQS 原理？

volatile state + CLH 队列 + 模板方法。通过 CAS 修改 state，失败则入队阻塞。

2. ReentrantLock 非公平锁和公平锁的区别？

非公平锁先 CAS 抢锁，抢不到再排队；公平锁先检查队列是否有前驱，有则排队。

3. AQS 等待队列的数据结构？

CLH 双向链表，每个节点包含线程、等待状态、前驱/后继指针。

4. Condition 的 await 和 signal 原理？

await 释放锁、加入条件队列、阻塞；signal 将节点从条件队列移到同步队列。

5. 独占锁和共享锁的区别？

独占锁（ReentrantLock）：一次一个线程持有。共享锁（Semaphore/CountDownLatch）：可多个线程同时持有。

6. ReentrantLock 和 synchronized 选哪个？

锁升级不复杂的场景用 synchronized（简洁）；需要超时、可中断、公平锁、多 Condition 时用 ReentrantLock。
