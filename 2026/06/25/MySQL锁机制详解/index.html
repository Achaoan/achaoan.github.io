六、面试题

1. Record Lock、Gap Lock、Next-Key Lock 的区别？

Record Lock 锁记录，Gap Lock 锁间隙（防插入），Next-Key Lock = Record + Gap（RR 默认，防幻读）。

2. 间隙锁在什么隔离级别生效？

REPEATABLE READ。READ COMMITTED 级别下间隙锁失效，可能导致幻读。

3. 什么是死锁？怎么排查？

两个事务互相持有对方需要的锁。用 SHOW ENGINE INNODB STATUS 查看，或查 information_schema.INNODB_TRX。

4. 无索引更新为什么会导致锁表？

无索引时 InnoDB 需要全表扫描，所有行都可能被更新，最终升级为表锁。

5. MVCC 和锁的关系？

MVCC 用于快照读（无锁），锁用于当前读（FOR UPDATE/UPDATE/DELETE）。

6. 如何避免死锁？

固定访问顺序、缩短事务、使用 RC 级别、通过索引减少锁范围。
