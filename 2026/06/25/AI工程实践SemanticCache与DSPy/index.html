三、DSPy（声明式 AI 编程框架）

是什么

DSPy 是一种新的 AI 编程范式，用声明式编程替代 Prompt Engineering。核心思想是：用 Python 模块描述 AI 任务，由优化器自动找到最佳的 Prompt 配置。

五、面试题

1. Semantic Cache 的原理？

基于语义相似度缓存 LLM 回答。将请求转为 Embedding 向量，新请求与缓存向量计算相似度，超过阈值则直接返回缓存结果。命中率高，可降低 80% 延迟和成本。

2. 合成数据的作用和风险？

解决标注成本高、隐私限制、数据不平衡问题。风险是 Model Collapse（用合成数据训练的模型质量逐代下降）和放大偏见。解决方案是真实数据 + 合成数据混合。

3. DSPy 和传统 Prompt Engineering 的区别？

DSPy 用声明式编程替代手写 Prompt。定义签名和模块，由优化器自动搜索最佳 Prompt 配置。换模型时只需重新优化，不需要重写代码。

4. AI 工程化的核心要点？

成本控制（语义缓存、模型分级）、质量控制（RAGAS 评估、幻觉检测）、数据管理（合成数据）、开发效率（DSPy 自动化）、安全合规（Guardrails）。

5. 什么是 Model Collapse？

用合成数据训练的模型质量逐代下降的现象。类似复印机反复复印，越来越模糊。原因是合成数据丢失了真实数据的多样性。解决方案是混合真实数据和合成数据。
