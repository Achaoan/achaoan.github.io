九、面试题

1. 三次握手过程？

SYN → SYN+ACK → ACK。同步序列号，确认双方收发能力，防止历史连接。

2. 为什么是三次不是两次？

防止历史 SYN 建立无效连接。两次握手时，服务端收到旧 SYN 就会建立连接并分配资源。

3. 四次挥手过程？

FIN → ACK → FIN → ACK。TIME_WAIT 2MSL 保证最后一个 ACK 到达。

4. TIME_WAIT 的作用？

保证 ACK 到达（防止服务端重发 FIN），让旧报文在网络中消失。

5. 流量控制和拥塞控制的区别？

流量控制解决收发速度不匹配（滑动窗口），拥塞控制解决网络链路过载（慢启动、拥塞避免、快速恢复）。

6. 如何解决粘包？

固定长度、分隔符、长度字段前缀。Netty 中 LengthFieldBasedFrameDecoder 最常用。
