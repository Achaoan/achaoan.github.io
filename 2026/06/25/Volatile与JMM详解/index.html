一、Java 内存模型（JMM）

什么是 JMM？

JMM（Java Memory Model）定义了多线程并发访问共享内存的规范，解决了可见性、有序性、原子性三大问题。

内存模型结构

JMM 的 8 种操作

操作 作用
lock 锁定主内存变量，独占
unlock 解锁
read 从主内存读到工作内存
load 从工作内存副本载入线程
use 线程使用变量值
assign 线程赋值给变量
store 从工作内存写到主内存
write 主内存写入变量

二、三大特性

1. 原子性

一个或多个操作要么全部执行且不被打断，要么全部不执行。

解决：synchronized、Lock、Atomic 类

2. 可见性

一个线程修改共享变量后，其他线程能立即看到。

解决：volatile、synchronized（unlock 前写回主内存）、final（构造完成后可见）

3. 有序性

编译器和处理器可能对指令重排序，但不影响单线程结果。

解决：volatile（禁止重排序）、synchronized、Lock

四、Happens-Before 规则

什么是 Happens-Before？

如果 A happens-before B，则 A 的操作结果对 B 可见，且 A 的执行顺序在 B 之前。

8 条规则

规则 说明
程序次序规则 一个线程内，书写在前面的操作 happens-before 后面的
volatile 规则 volatile 变量的写 happens-before 后续的读
锁规则 unlock happens-before 后续的 lock
线程启动规则 Thread.start() happens-before 该线程的任何操作
线程终止规则 线程的所有操作 happens-before 其他线程检测到该线程终止
线程中断规则 调用 interrupt() happens-before 检测到中断事件
对象终结规则 对象的构造函数结束 happens-before finalize()
传递性 A happens-before B，B happens-before C → A happens-before C

五、面试题

1. volatile 能保证原子性吗？

不能。count++ 是读-改-写三步操作，volatile 只保证可见性和有序性。原子性需要 synchronized 或 Atomic 类。

2. volatile 和 synchronized 的区别？

volatile 是轻量级同步，只保证可见性和有序性，不阻塞线程；synchronized 保证三大特性，会阻塞线程。

3. JMM 是什么？

Java 内存模型，定义线程和主内存的交互规范，解决可见性、有序性、原子性问题。

4. Happens-Before 规则有哪些？

8 条规则，最重要的是 volatile 规则、锁规则和传递性。

5. DCL 单例为什么需要 volatile？

防止指令重排序导致其他线程拿到未初始化完成的对象。

6. 内存屏障是什么？

禁止特定类型的重排序，volatile 底层通过内存屏障实现可见性和有序性。
