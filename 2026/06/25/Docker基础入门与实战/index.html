一、Docker 核心概念

什么是 Docker？

Docker 是一个开源的应用容器引擎，让开发者可以打包应用及依赖到一个轻量级、可移植的容器中。

三大核心概念

概念 说明 类比
镜像（Image） 只读模板，包含运行环境和配置 类（Class）
容器（Container） 镜像的运行实例，可读写 实例（Instance）
仓库（Repository） 集中存储镜像的地方 GitHub

三、Dockerfile

常用指令

指令 说明 示例
FROM 基础镜像 FROM openjdk:17-slim
WORKDIR 工作目录 WORKDIR /app
COPY 复制文件到镜像 COPY target/app.jar app.jar
ADD 复制（支持 URL/解压） ADD app.tar.gz /app
RUN 构建时执行命令 RUN apt-get update && apt-get install -y curl
ENV 环境变量 ENV JAVA_OPTS="-Xms512m -Xmx512m"
EXPOSE 暴露端口（文档说明） EXPOSE 8080
CMD 容器启动命令 CMD ["java", "-jar", "app.jar"]
ENTRYPOINT 容器入口点 ENTRYPOINT ["java", "-jar", "app.jar"]

七、面试题

1. 镜像和容器的区别？

镜像是只读模板，容器是镜像的运行实例。类比：镜像 = 类，容器 = 对象。

2. CMD 和 ENTRYPOINT 的区别？

CMD 提供默认命令，可被覆盖。ENTRYPOINT 是容器入口点，不可被覆盖。通常组合使用：ENTRYPOINT 固定程序，CMD 提供默认参数。

3. 如何减小 Docker 镜像体积？

  • 使用 Alpine 基础镜像
  • 多阶段构建
  • 合并 RUN 命令
  • 清理 apt/yum 缓存

4. Docker 和虚拟机的区别？

Docker 共享宿主机内核，资源占用小，启动秒级；虚拟机包含完整 OS，资源占用大，启动分钟级。

5. docker-compose 的作用？

定义和运行多容器应用，通过 YAML 配置管理多个服务、网络和卷。

6. 如何查看容器日志和进入容器？

docker logs -f <container> 查看日志，docker exec -it <container> /bin/bash 进入容器。
