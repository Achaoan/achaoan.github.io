大语言模型（LLM）基础：Transformer、Token、上下文窗口

一、什么是大语言模型

大语言模型（Large Language Model, LLM）是一种基于深度学习的神经网络模型，通过海量文本数据训练，能够理解和生成人类语言。

二、Transformer 架构

现代 LLM 都基于 Transformer 架构（Google 2017 年论文《Attention Is All You Need》），核心创新是自注意力机制。

Self-Attention 详解

三、Token（词元）

Token 是模型处理文本的最小单位，不是按”字”或”词”简单切分。

四、上下文窗口（Context Window）

上下文窗口是模型一次能”看到”的最大 Token 数，决定了能处理多长的内容。

五、参数（Parameters）

模型的大小用参数量衡量，参数是神经网络中可学习的权重。

七、面试题

1. LLM 的”理解”和人类的”理解”有什么不同？

LLM 不是真正理解语言，而是基于统计规律做概率预测。它从前缀推断后缀，不拥有意识或真正的语义理解能力。但高质量的预训练让它能”模拟”理解。

2. Transformer 的核心创新是什么？

Self-Attention 机制，让模型能一次性”看到”所有 Token 并计算它们之间的关系。相比 RNN 的串行处理，Transformer 可并行训练，这是 GPT 能规模化的根本原因。

3. Temperature 的作用是什么？

控制输出的随机性。低温度（0-0.3）输出确定性强，适合代码和数学。高温度（0.8-1.0）输出多样，适合创意写作。Temperature=0 时完全确定。

4. 上下文窗口越大越好吗？

不完全是。大窗口意味着能处理更长内容，但存在 Lost in the Middle 问题（中间信息容易被忽略），且计算成本随窗口大小平方增长。RAG 仍是处理超长文本的更好方案。

5. Token 是什么？为什么重要？

Token 是模型处理文本的最小单位。计费、上下文限制都基于 Token 数。英文约 1 词=1.3 tokens，中文约 1 字=1-2 tokens，中文比英文更”贵”。
