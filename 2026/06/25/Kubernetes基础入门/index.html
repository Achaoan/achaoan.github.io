Kubernetes基础入门：Pod、Deployment、Service核心概念

本文最后更新于：2 小时前

一、Kubernetes 概述

是什么？

Kubernetes（K8s）是 Google 开源的容器编排平台，用于自动部署、扩展和管理容器化应用。

二、核心概念

架构

Master 节点组件

组件 说明
kube-apiserver 所有操作的入口，提供 REST API
kube-scheduler 将 Pod 分配到合适的 Node
kube-controller-manager 运行各种控制器（Deployment、Node、Service等）
etcd 集群状态存储（键值对，基于 Raft）

Worker 节点组件

组件 说明
kubelet 管理 Pod 的生命周期，与 apiserver 通信
kube-proxy 实现 Service 的网络代理和负载均衡
容器运行时 Docker、containerd、CRI-O

三、Pod

什么是 Pod？

K8s 最小的调度单元，一个 Pod 包含一个或多个容器（共享网络和存储）。

五、Service

什么是 Service？

为一组 Pod 提供稳定的网络入口和负载均衡。

Service 类型

类型 说明 访问方式
ClusterIP（默认） 集群内部 IP，外部不可访问 集群内访问

九、面试题

1. K8s 核心组件有哪些？

Master：apiserver、scheduler、controller-manager、etcd。Worker：kubelet、kube-proxy、容器运行时。

2. Pod 是什么？

最小调度单元，包含一个或多个容器，共享网络和存储。

3. Deployment 的作用？

管理 Pod 副本，支持滚动更新、回滚、扩缩容，声明式更新应用。

4. Service 的作用？

为 Pod 提供稳定的网络入口和负载均衡，支持 ClusterIP/NodePort/LoadBalancer 三种类型。

5. Pod 的生命周期？

Pending → Running → Succeeded/Failed。

6. K8s 和 Docker 的关系？

Docker 是容器运行时，K8s 是容器编排平台。K8s 管理 Docker 容器的部署、扩展和调度。
