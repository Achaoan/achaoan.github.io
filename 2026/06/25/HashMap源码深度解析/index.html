六、JDK 7 vs JDK 8 对比

对比项 JDK 7 JDK 8
数据结构 数组 + 链表 数组 + 链表 + 红黑树

七、常见问题

为什么容量是 2 的幂？

  1. (n - 1) & hash 等价于 hash % n，位运算更快
  2. 扩容时数据分布均匀，只需看高位 bit

HashMap 线程安全吗？

不安全。并发场景：

  • JDK 7：死循环（头插法导致环形链表）
  • JDK 8：数据丢失、size 不准确
  • ✅ 解决方案：ConcurrentHashMap

为什么默认负载因子是 0.75？

时间和空间的权衡：

  • 太高（如 1）：空间利用率高，但碰撞概率大，查询慢
  • 太低（如 0.5）：碰撞少，但浪费空间

0.75 是官方经过大量测试后的最优平衡值。

八、面试题

1. HashMap put 流程？

计算 hash → 找下标 → 空则直接插 → 碰撞则遍历链表/红黑树 → 查重覆盖或新增 → 判断扩容。

2. 扩容机制？

翻倍扩容，JDK 8 用高位 bit 判断数据位置，避免 rehash。

3. 为什么转红黑树阈值是 8？

泊松分布，链表长度 8 的概率极低，选择 8 平衡性能和空间。

4. JDK 7 头插法有什么问题？

并发扩容时形成环形链表，导致死循环。JDK 8 改尾插法解决。

5. HashMap 线程安全吗？替代方案？

不安全。用 ConcurrentHashMap 或 Collections.synchronizedMap()。
