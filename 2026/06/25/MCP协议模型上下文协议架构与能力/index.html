MCP 模型上下文协议：架构、三种能力与 Skill 概念

本文最后更新于：37 分钟前

一、什么是 MCP

MCP（Model Context Protocol）是 Anthropic 在 2024 年底提出的开放协议，用于标准化 AI 模型与外部工具/数据源的连接方式。可以理解为”AI 界的 USB-C”。

三、三种核心能力

MCP Server 可以暴露三种类型的资源给 AI 模型：

1. Tools（工具）—— 让 AI 执行操作

2. Resources（资源）—— 让 AI 读取数据

3. Prompts（提示模板）—— 让 AI 按模板工作

四、Skill（技能）

在 AI 平台中，Skill 是一组预定义的指令 + 工具配置，让 AI 具备完成特定类型任务的专业能力。

五、MCP vs Plugin / Function Calling

六、面试题

1. MCP 解决的核心问题是什么？

统一 AI 模型与外部工具/数据源的连接标准。之前每个框架各自定义工具接口，工具不可复用。MCP 让工具即插即用，一次开发任意平台可用。

2. MCP 的三层架构？

Host（AI 应用如 Claude Code）→ Client（与 Server 一对一连接）→ Server（暴露工具/资源/模板）。每个 Server 专注一个领域。

3. MCP Server 的三种能力？

Tools（可调用的函数，对应行动能力）、Resources（可读取的数据，对应知识能力）、Prompts（预定义的对话模板，对应规范能力）。

4. Skill 是什么？

一组预定义的指令 + 工具配置，让 AI 成为领域专家。Skill 是专业化的能力包，可复用分享，激活后持续影响 AI 行为。

5. MCP 和 Function Calling 的关系？

Function Calling 是能力（模型可以调用函数），MCP 是标准协议（规范了如何暴露和调用函数）。用 MCP 协议暴露工具，底层用 Function Calling 能力执行。
