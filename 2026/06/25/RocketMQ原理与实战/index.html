八、面试题

1. RocketMQ 的架构组件？

NameServer（路由）、Broker（存储）、Producer、Consumer、Topic、MessageQueue。

2. RocketMQ 事务消息的原理？

半消息 → 执行本地事务 → 确认提交/回滚。Broker 回调查询接口，实现最终一致性。

3. RocketMQ 顺序消息怎么实现？

Producer 将相同顺序 ID 的消息发到同一个 MessageQueue，Consumer 单线程消费该 Queue。

4. RocketMQ 怎么保证消息不丢失？

同步刷盘 + 同步复制 + 重试机制 + 死信队列。

5. RocketMQ 和 Kafka 的适用场景？

RocketMQ：业务消息、事务消息、可靠投递。Kafka：日志收集、大数据、高吞吐场景。
