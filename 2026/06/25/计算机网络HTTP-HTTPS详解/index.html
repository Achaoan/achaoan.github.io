一、HTTP 协议概述

什么是 HTTP？

HTTP（HyperText Transfer Protocol）是无状态、无连接的应用层协议，基于 TCP 实现。

八、面试题

1. HTTP/1.1 和 HTTP/2 的区别？

HTTP/2 是二进制分帧、多路复用、头部压缩（HPACK）、服务端推送。解决了 HTTP/1.1 的队头阻塞问题。

2. HTTPS 握手过程？

ClientHello → ServerHello + 证书 → 交换密钥（非对称）→ 协商对称密钥 → 加密通信。

3. 为什么 HTTPS 使用混合加密？

非对称加密慢但安全交换密钥，对称加密快但密钥分发困难。混合加密结合两者优点：非对称加密交换对称密钥，对称加密传输数据。

4. TLS 1.2 和 1.3 的区别？

TLS 1.3 握手 1-RTT（1.2 是 2-RTT），支持 0-RTT 恢复，移除不安全算法，强制前向安全。

5. HTTP 状态码 301 和 302 的区别？

301 永久重定向（浏览器缓存后续请求），302 临时重定向（每次请求仍然访问原 URL）。

6. DNS 解析过程？

先查浏览器缓存 → 操作系统缓存 → 本地 DNS → 根 → 顶级域 → 权威服务器。

7. Cookie 和 Session 的区别？

Cookie 存客户端（每次请求自动携带），Session 存服务端内存/Redis。Cookie 不安全，Session 需要存储空间。分布式架构常用 JWT 替代 Session。
