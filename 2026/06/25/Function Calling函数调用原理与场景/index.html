Function Calling（函数调用）：原理、流程与应用场景

本文最后更新于：38 分钟前

一、什么是 Function Calling

Function Calling（函数调用）是 LLM 的一种能力——模型在回答中可以主动请求调用外部工具/函数，并将工具返回的结果整合进回答中。

六、面试题

1. Function Calling 的本质是什么？

LLM 输出结构化的函数调用请求，外部系统执行函数，结果返回给模型生成回答。模型不执行函数，模型请求函数被调用。

2. Function Calling 的典型流程？

用户输入 → LLM 判断需要调用工具 → 输出结构化请求 → 外部系统执行 → 结果返回 → LLM 生成回答。

3. Function Calling 和 MCP 的关系？

Function Calling 是能力，MCP 是标准协议。MCP 标准化了 Function Calling 的实现方式。使用 MCP 时底层仍需要 Function Calling 能力。

4. Function Calling 的安全注意事项？

写操作必须用户确认，参数需要严格 schema 限制，工具数量不宜过多，设置超时机制，定期审计调用日志。
