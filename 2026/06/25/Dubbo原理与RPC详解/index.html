四、负载均衡策略

策略 说明 默认权重
Random（默认） 随机，按权重分配 权重越大概率越高
RoundRobin 轮询，平滑加权轮询 按权重比例分配
LeastActive 最少活跃调用数 慢的节点接收请求少
ConsistentHash 一致性哈希 相同参数发到相同节点

五、集群容错

策略 说明 适用场景
Failover（默认） 失败重试（换节点） 幂等操作（查询）
Failfast 立即失败 非幂等操作（新增）
Failsafe 忽略失败 日志上报
Failback 异步重试 通知类场景
Forking 并行调用多个，一个返回即返回 实时性高的查询
Broadcast 广播所有，全部返回即成功 缓存预热

六、协议对比

协议 优点 缺点 适用场景
Dubbo 长连接，NIO，高吞吐 连接数多 小数据量高并发
HTTP 通用，穿透防火墙 性能低 异构系统对接
REST RESTful 风格 性能一般 开放 API
gRPC 流式调用，跨语言 需定义 proto 高性能跨语言
Thrift 跨语言，性能高 配置复杂 跨语言调用

八、面试题

1. Dubbo 的调用流程？

Consumer 从注册中心获取服务地址列表，通过负载均衡选一个 Provider，基于 Netty 长连接远程调用。

2. 负载均衡策略有哪些？

Random（默认）、RoundRobin、LeastActive、ConsistentHash。

3. 集群容错策略有哪些？

Failover（重试）、Failfast（快速失败）、Failsafe（忽略失败）、Failback（异步重试）、Forking（并行调用）。

4. Dubbo SPI 和 Java SPI 的区别？

Java SPI 一次性加载所有实现，无法按需获取。Dubbo SPI 可以按需加载、支持 AOP 包装、支持依赖注入。

5. Dubbo 支持哪些协议？

Dubbo（默认高性能）、HTTP、REST、gRPC、Thrift。
