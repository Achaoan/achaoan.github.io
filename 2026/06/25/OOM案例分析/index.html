1. OOM 排查步骤？

看日志确认类型 → 启用在 OOM 时自动 dump → 用 MAT 分析堆转储 → 定位泄漏引用链 → 修复。

2. 常见 OOM 类型及原因？

Heap space（对象太多）、Metaspace（类太多）、Direct buffer memory（DirectByteBuffer 泄漏）、Unable to create thread（线程数超限）。

3. 堆转储用什么工具分析？

MAT（Memory Analyzer Tool）、JProfiler、VisualVM。

4. 内存泄漏和内存溢出的区别？

泄漏：对象无法被回收，可用内存越来越少。溢出：内存不够分配新对象。泄漏是溢出的常见原因。

5. 频繁 Full GC 怎么排查？

jstat 看老年代/元空间使用趋势 → jmap 看对象分布 → 堆转储分析泄漏疑点。
