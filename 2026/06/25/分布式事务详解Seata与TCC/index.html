八、面试题

1. 分布式事务有哪些方案？

2PC（XA）、TCC、Seata AT、可靠消息最终一致性、最大努力通知。

2. TCC 的 Try/Confirm/Cancel 分别做什么？

Try 预占资源，Confirm 确认执行，Cancel 回滚释放。

3. Seata AT 模式的原理？

执行 SQL 时自动记录 undo-log，全局事务回滚时通过 undo-log 逆向补偿。

4. 可靠消息最终一致性的实现思路？

本地消息表+MQ：业务操作和消息记录在同一个本地事务，异步发送消息，消费者消费成功后手动确认。

5. 2PC 的缺点？

同步阻塞、单点故障、数据不一致风险。

6. TCC 的空回滚和幂等怎么处理？

空回滚：Try 未执行就收到 Cancel，需要识别并忽略。幂等：用全局事务 ID + 状态表，保证 Confirm/Cancel 只生效一次。
