八、面试题

1. Spring Security 的认证流程？

UsernamePasswordAuthenticationFilter → AuthenticationManager → ProviderManager → DaoAuthenticationProvider → UserDetailsService → PasswordEncoder 校验 → 设置 SecurityContext。

2. 过滤器链顺序是怎样的？

从 SecurityContextPersistenceFilter → LogoutFilter → 登录过滤器 → AnonymousFilter → ExceptionTranslationFilter → FilterSecurityInterceptor。

3. @PreAuthorize 和 @Secured 的区别？

@PreAuthorize：支持 SpEL 表达式，功能更强大。@Secured：只能指定角色名，功能简单。

4. JWT 认证和 Session 认证的优缺点？

JWT 无状态，扩展性好，但不方便撤销；Session 有状态，方便管理，但需要共享存储。

5. Spring Security 如何防止 CSRF？

生成 CSRF Token 绑定在 Session 中，请求时校验。REST API 通常禁用。

6. 如何实现动态权限（URL-角色可配置）？

自定义 FilterInvocationSecurityMetadataSource，从数据库加载 URL-角色映射，返回给 AccessDecisionManager 决策。
