一、Redis 持久化

RDB（快照）

将内存数据在某个时间点保存到磁盘的二进制快照。

优点：文件小，恢复快，适合备份
缺点：可能丢失两次快照间的数据

AOF（Append Only File）

记录每次写操作命令，重启时重新执行恢复。

优点：数据安全性高（最多丢 1 秒数据）
缺点：文件大，恢复慢

混合持久化（Redis 4.0+）

哨兵模式（Sentinel）

四、缓存策略

过期策略

策略 说明
定时删除 创建过期 key 时启动定时器，到期删除（CPU 开销大）
惰性删除 访问时检查是否过期，过期则删除（内存浪费）
定期删除（Redis 使用） 每隔 100ms 随机抽查一批过期 key 删除

内存淘汰策略

策略 说明
noeviction 不淘汰，写入时报错
allkeys-lru（推荐） 全体 key 中淘汰最近最少使用的
volatile-lru 设置过期时间的 key 中淘汰最近最少使用的
allkeys-random 全体 key 随机淘汰
volatile-random 过期 key 随机淘汰
volatile-ttl 过期 key 中淘汰即将过期的

缓存问题

缓存一致性

方案 说明
先更新数据库，再删除缓存（推荐） 最终一致性，删除失败可异步重试
先删除缓存，再更新数据库 并发下可能读到旧数据
延时双删 更新后延迟再删一次
订阅 binlog 同步 Canal + MQ 异步同步，解耦最彻底

六、面试题

1. RDB 和 AOF 的区别和选型？

RDB：文件小恢复快，可能丢数据；AOF：数据安全，文件大恢复慢。Redis 4.0+ 推荐混合持久化。

2. 哨兵模式和 Cluster 模式怎么选？

哨兵：主从 + 高可用，数据都在主库（容量受单机限制）。Cluster：分片存储，可水平扩展，适合大数据量。

3. 缓存穿透/击穿/雪崩的区别和解决方案？

穿透：查不存在的数据 → 布隆过滤器。击穿：热点 key 过期 → 互斥锁。雪崩：大量 key 过期 → 过期时间加随机值。

4. Redis 分布式锁如何实现？

SETNX + 过期时间（单节点），RedLock（多节点），Redisson 封装了看门狗自动续期。

5. 如何保证缓存和数据库一致性？

先更新数据库再删缓存，配合 Canal 订阅 binlog 异步同步。

6. Redis 的过期策略和淘汰策略？

过期策略：定期删除 + 惰性删除。淘汰策略：推荐 allkeys-lru。
