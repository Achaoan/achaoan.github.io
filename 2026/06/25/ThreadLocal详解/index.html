七、面试题

1. ThreadLocal 原理？

每个 Thread 持有 ThreadLocalMap，key 是 ThreadLocal 弱引用，value 是存储的值。通过 set() / get() 操作当前线程的 Map。

2. ThreadLocal 内存泄漏原因？

Entry 的 key 是弱引用，GC 后被回收，但 value 是强引用。线程池中线程复用，value 无法被回收。**必须 remove()**。

3. 为什么用弱引用 key？

防止 ThreadLocal 对象无法被 GC。如果用强引用，即使业务不再使用 ThreadLocal，它也无法被回收。

4. ThreadLocalMap 的哈希冲突怎么解决？

线性探测（开放地址法），因为 ThreadLocalMap 的 Entry 数量通常很少。

5. InheritableThreadLocal 有什么用？有什么坑？

子线程继承父线程的 ThreadLocal 值。在线程池中会失效（线程复用），可用阿里 TTL 解决。

6. ThreadLocal 的应用场景？

Spring 事务管理器（事务上下文）、请求上下文（用户信息）、MDC 日志、SimpleDateFormat 线程安全。
