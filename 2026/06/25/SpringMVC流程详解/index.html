Spring MVC请求流程详解：DispatcherServlet、HandlerMapping、HandlerAdapter

本文最后更新于：8 分钟前

一、Spring MVC 核心组件

组件 说明
DispatcherServlet 前端控制器，请求入口，统一分发
HandlerMapping 处理器映射器，根据 URL 找到对应的 Handler
HandlerAdapter 处理器适配器，按规则调用 Handler
Handler / Controller 业务控制器，处理具体请求
ViewResolver 视图解析器，将逻辑视图名解析为真实视图
View 视图对象，渲染响应结果
ModelAndView 数据和视图的封装

二、Spring MVC 请求处理流程（9步）

详细步骤

第 1 步： 客户端发送 HTTP 请求，DispatcherServlet 接收。
第 2 步： DispatcherServlet 调用 HandlerMapping 根据 URL 查找对应的 Handler（及拦截器），返回 HandlerExecutionChain。
第 3 步： DispatcherServlet 根据 Handler 类型选择合适的 HandlerAdapter。
第 4 步： 执行拦截器 preHandle() 方法。
第 5 步： HandlerAdapter 调用 Handler 的方法（如 Controller 的方法），执行完毕后返回 ModelAndView。
第 6 步： 执行拦截器 postHandle() 方法。
第 7 步： DispatcherServlet 调用 ViewResolver 将逻辑视图名解析为物理视图。
第 8 步： 视图渲染，将 Model 数据填入视图。
第 9 步： 执行拦截器 afterCompletion() 方法，返回响应。

四、常用的注解

注解 作用
@Controller 标识控制器类
@RestController @Controller + @ResponseBody
@RequestMapping 映射 URL
@GetMapping / @PostMapping 简化 HTTP 方法映射
@RequestParam 绑定请求参数
@PathVariable 绑定 URL 路径变量
@RequestBody 绑定请求体 JSON
@ResponseBody 直接返回数据（不解析视图）
@ResponseStatus 指定响应状态码

六、拦截器 vs 过滤器

对比 Filter Interceptor
