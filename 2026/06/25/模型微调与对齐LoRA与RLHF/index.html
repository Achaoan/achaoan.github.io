模型微调与对齐：LoRA、RLHF、DPO 原理与实践

本文最后更新于：20 分钟前

一、什么是微调

微调（Fine-tuning）是在预训练模型的基础上，用特定领域数据进一步训练，让模型适应特定任务。

二、LoRA（低秩适配）

LoRA（Low-Rank Adaptation）是一种参数高效的微调方法，只训练少量新增参数。

三、RLHF（基于人类反馈的强化学习）

RLHF（Reinforcement Learning from Human Feedback）是让模型行为符合人类偏好的训练方法。

四、DPO（直接偏好优化）

DPO（Direct Preference Optimization）是 RLHF 的简化替代方案。

五、面试题

1. LoRA 的原理？

冻结原始权重，插入两个小矩阵 A×B（低秩分解）。训练时只更新 A 和 B，推理时合并回原始权重，零额外开销。参数量减少 10000 倍。

2. LoRA 的 rank 参数怎么选？

r=8 适合简单任务，r=16 标准设置，r=64 适合复杂任务。rank 越大适配能力越强但参数量也越大。target_modules 通常选 Q 和 V 矩阵即可。

3. QLoRA 是什么？

QLoRA = Quantization + LoRA。先将模型量化到 4-bit 再做 LoRA 微调。单张 24GB 显卡可微调 70B 模型，将微调成本从万元级降到几百元级。

4. RLHF 的三步流程？

收集人类偏好数据 → 训练奖励模型 → PPO 优化 LLM。流程复杂（3-4 个模型），不稳定，且需要大量人工标注。Reward Hacking 是主要风险。

5. DPO 和 RLHF 的区别？

DPO 不需要奖励模型和 PPO，直接用偏好数据优化 LLM。实现简单、训练稳定，效果接近或达到 RLHF。DPO 正在成为新一代模型对齐的主流方法。
