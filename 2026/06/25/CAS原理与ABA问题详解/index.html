CAS原理与ABA问题解决方案

本文最后更新于：2 小时前

一、CAS 概述

什么是 CAS？

CAS（Compare And Swap）是一种乐观锁技术，通过比较内存值是否等于预期值，相等则更新，不相等则重试。

硬件支持

CAS 是CPU 原语（不可中断），在 x86 上对应 CMPXCHG 指令。

原子性保证：由 CPU 硬件保证比较和交换的原子性。

七、synchronized vs CAS

对比 synchronized CAS
类型 悲观锁 乐观锁

八、面试题

1. CAS 原理？

CPU 原语 CMPXCHG，比较内存值是否等于预期值，相等则更新，不相等则重试。

2. CAS 的三大问题？

ABA 问题（AtomicStampedReference 解决）、自旋开销大（LongAdder 分散）、只能保证一个变量（用 AtomicReference 封装）。

3. ABA 问题及解决？

值被修改后又改回原值。用 AtomicStampedReference（版本号）解决。

4. CAS 自旋一定会阻塞吗？

不会，CAS 在用户态自旋（循环），不会进入内核态。但长时间自旋会浪费 CPU。

5. LongAdder 怎么解决 CAS 竞争？

分散到 Cell[] 数组，每个线程 CAS 自己的 Cell，求和时汇总（sum = base + ΣCell[]）。

6. 乐观锁和悲观锁的区别？

悲观锁（synchronized）：假设冲突一定发生，加锁阻塞。乐观锁（CAS）：假设冲突少，失败则重试。
