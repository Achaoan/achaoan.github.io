六、面试题

1. 短链生成的 ID 怎么保证唯一？

号段模式（DB 预取一批 ID），雪花算法，或用 Redis incr。

2. 短链 Key 怎么生成？

数字 ID → Base62 编码（0-9a-zA-Z），6 位支持 568 亿条。

3. 301 和 302 怎么选？

302 临时重定向，方便统计点击和做安全校验。

4. 短链服务的高性能设计？

本地缓存 Caffeine + Redis 两级缓存，预加载热点数据，无状态水平扩展。

5. 短链冲突怎么处理？

MD5 截取方案可能冲突，解决：第一次冲突时末尾加随机字符再次尝试。
号段模式 + Base62 编码不会冲突（ID 唯一）。
