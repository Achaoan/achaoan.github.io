RAG 检索增强生成：原理、工作流与核心组件

本文最后更新于：22 分钟前

一、什么是 RAG

RAG（Retrieval-Augmented Generation）是一种让 LLM 能够访问外部知识库的技术方案。它通过检索相关文档片段来增强 LLM 的回答质量。

二、RAG 标准工作流程

三、核心组件详解

1. Embedding Model（嵌入模型）

将文本转为向量（固定维度的浮点数数组），语义相近的文本向量距离也近。

2. Vector Database（向量数据库）

专门用于存储和检索向量的数据库。

3. Chunking Strategy（分块策略）

将文档切成适当大小的块，是 RAG 效果的关键因素。

4. Retrieval Strategy（检索策略）

四、RAG vs 微调

五、面试题

1. RAG 解决了什么问题？

LLM 知识过时、不知道私有数据、产生幻觉。RAG = 给 LLM 配参考书，通过外部检索增强回答质量。

2. RAG 的三阶段流程？

预处理（文档切分 + 向量化 + 存入向量库）→ 查询（问题向量化 + 相似度检索）→ 生成（问题 + 检索结果 → LLM 回答）。

3. Chunking 策略怎么选？

固定大小分块简单但可能切断语义，语义分块保留完整性但实现复杂，重叠分块避免边界切断问题。推荐 256-1024 tokens + 10-20% 重叠。

4. RAG 和微调的区别？

RAG 不修改模型，通过检索增强知识，适合实时更新场景。微调修改模型参数，适合改变行为风格。两者可结合使用。

5. Embedding Model 的作用？

将文本转为向量，语义相近的文本向量距离也近。RAG 中用同一个 Embedding 模型将问题和文档都向量化，通过余弦相似度匹配。
