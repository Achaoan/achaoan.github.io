五、为什么需要三级缓存？

只用两级不行吗？

只用一级缓存：

  • 半成品 Bean 直接暴露在一级缓存
  • 其他线程 getBean 可能拿到未初始化完成的 Bean
  • 混乱：成品和半成品混在一起

只用两级缓存（去掉三级）：

  • 无法解决 AOP 代理问题
  • 如果 A 需要 AOP 代理，B 依赖的 A 必须是代理对象
  • 没有三级缓存，无法在实例化后立即产生代理对象

三级缓存的必要性

核心设计思想

用三级缓存延迟 AOP 代理的创建，只有出现循环依赖时才触发。

七、面试题

1. Spring 怎么解决循环依赖？

三级缓存：一级存成品 Bean，二级存半成品 Bean，三级存 ObjectFactory。A 先实例化放入三级缓存，B 依赖 A 时从三级缓存获取早期引用，B 完成后再完成 A 的初始化。

2. 为什么需要三级缓存，两级不行吗？

两级缓存无法处理 AOP 代理。三级缓存的 ObjectFactory 只有在循环依赖时才触发 AOP 代理创建，避免正常流程中提前代理。

3. 哪些循环依赖解决不了？

构造器注入（实例化时需要参数）和非单例 Bean（prototype 不缓存）解决不了。

4. 三级缓存和 AOP 的关系？

三级缓存的 getEarlyBeanReference 会调用 SmartInstantiationAwareBeanPostProcessor 提前创建 AOP 代理，确保依赖方拿到的是代理对象。

5. 构造器注入的循环依赖怎么解决？

用 @Lazy 注入代理，或改用字段注入 / setter 注入。
