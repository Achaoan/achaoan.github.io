六、面试题

1. Redis 分布式锁怎么实现？

SETNX key value EX 30 + Lua 脚本释放锁。Redissson 封装了看门狗自动续期。

2. Redisson 看门狗的原理？

加锁后启动定时任务（每 10s），检测业务是否还在执行，是则续期到 30s。

3. ZK 分布式锁的原理？

临时顺序节点，最小序号获取锁，监听前一个节点释放。

4. Redis 锁和 ZK 锁的优缺点？

Redis 性能高，但可能锁丢失；ZK 强一致，但性能较低。选型看一致性要求。

5. 锁超时了业务还没完成怎么办？

Redisson 看门狗自动续期。或手动在业务中定期续期。

6. RedLock 解决了什么问题？

解决 Redis 主从切换导致的锁丢失问题：在多个独立节点上加锁，过半成功才算真正拿到锁。
