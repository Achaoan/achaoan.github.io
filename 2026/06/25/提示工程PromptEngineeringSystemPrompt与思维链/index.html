提示工程（Prompt Engineering）：System Prompt、Few-shot、思维链

本文最后更新于：39 分钟前

一、什么是提示工程

提示工程是通过设计输入给 LLM 的文本（Prompt），引导模型输出期望结果的工程技术。

二、System Prompt

System Prompt 是在对话开始前给模型的全局指令，设定角色、行为规则和输出约束。它贯穿整个对话始终。

三、Few-shot 与 Zero-shot

Zero-shot（零样本）

不给任何例子，直接提问。适合模型已经非常熟悉的任务。

Few-shot（少样本）

给出几个输入输出例子，让模型模仿模式。

四、Chain-of-Thought（思维链）

Chain-of-Thought（CoT）是让模型”一步一步思考”再给出答案的技术，能显著提高推理任务的准确率。

五、常用技巧

六、面试题

1. System Prompt 和 User Prompt 的区别？

System Prompt 是开发者设定的全局指令，贯穿整个对话，定义角色、行为规则和输出格式。User Prompt 是用户每次输入的具体问题。System Prompt 在 Attention 中占据更显著的位置。

2. Few-shot 为什么有效？

它为模型提供了输出格式的”锚点”和推理模式的参考。模型通过类比学习而不是仅依赖指令理解。3-5 个高质量例子是最佳数量。

3. Chain-of-Thought 的原理？

让模型先展示推理过程再给出结论，将隐式推理显式化。减少”跳步”导致的错误，让复杂问题被分解为简单子问题。Zero-shot CoT 只需加”让我们一步一步思考”。

4. 提示工程的核心原则是什么？

明确具体、结构化、提供上下文、定义约束。好的 Prompt = 目标 + 背景 + 格式 + 约束。
